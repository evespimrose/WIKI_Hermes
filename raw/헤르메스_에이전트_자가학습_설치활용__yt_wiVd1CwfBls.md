# 헤르메스 에이전트 — 자가학습 AI 에이전트 설치·활용

> 출처: [(FULL VER) 스스로 성장하는 AI가 나왔다.. 헤르메스 에이전트 사용법 A TO Z](https://www.youtube.com/watch?v=wiVd1CwfBls) — 그랜트 | 마케팅 AX, 2026-06-02

## 한 줄 요약
헤르메스(Hermes) 에이전트는 대화·작업을 거듭할수록 스스로 학습(셀프 임프루빙)해 사용자에게 맞춰지는 AI 에이전트로, Railway 서버에 원클릭 배포하고 OpenRouter·Telegram·Composio를 연동해 24시간 작동하는 '나만의 자비스'로 키우는 것이 핵심이다.

## 핵심 지식

### 헤르메스 에이전트의 정체성
- 메신저로 호출하면 사용자 PC를 돌아다니며 작업(연락·이메일 등)을 수행하는 에이전트(영상에서 비교 대상으로 든 '오픈클로'와 유사).
- 결정적 차이는 **자가 진화**: 오픈클로는 스스로 진화하지 않지만, 헤르메스는 대화를 통해 계속 발전한다. "나를 점차 알아가는 AI"를 만드는 것이 목적.

### 4대 메커니즘 (차별점)
- **셀프 임프루빙(학습 루프)**: 어떤 작업(예: 뉴스레터 생성)의 결과물이 만족스러우면, 에이전트가 그것을 `SKILL.md` 파일로 만들어 저장하고 사용자 피드백에 맞춰 계속 다듬는다. 모든 태스크를 스킬 파일화하려 한다.
- **성격 파일(`SOUL.md`)**: 에이전트의 성격·톤앤매너를 정의. 까칠/친절 등 조정 가능하고, 특정 인물을 모방시켜 상담역으로 쓸 수도 있다.
- **컨텍스트 파일**: `USER.md`(사용자 취향), `MEMORY.md`(프로젝트 전체 환경·맥락)를 지속 생성·적재. 에이전트는 "사용자가 누구인지" 파악하려고 이 MD들을 계속 갱신하는 습성이 있다.
- **상시 작동 + 메신저 연동**: VPS 서버에 올리면 24시간 작동(AI 직원 상주). Slack·Telegram 등과 연동(연동 난이도 때문에 영상에서는 Telegram 사용).

### 왜 서버가 필요한가
- 에이전트를 본인 PC에만 설치하면 PC가 꺼졌을 때 에이전트도 죽어 연락이 안 된다.
- 새벽·아침 등 아무 때나 메신저로 묻고 셀프 피드백을 받으려면, 24시간 켜져 있는 **서버를 빌려** 거기 설치해야 한다.

### 설치 — Railway 원클릭 배포
- **Railway**: 컴퓨터(서버) 한 대를 빌려주는 호스팅 플랫폼. 내부 세팅이 갖춰져 있고 GitHub 템플릿을 끌어와 바로 배포할 수 있어 비전공자에게 적합(AWS·Linux 직접 세팅의 병목을 우회).
- 절차: Railway 가입 → 결제(Hobby 플랜, 약 $5/월, **종량제** = 쓴 만큼 과금) → New → 하단 Templates → "헤르메스(Hermes)" 검색 → 가장 유명한 템플릿 선택 → Deploy to New Project → Config에서 admin username·password 입력 → Save → Deploy → 빌드 대기 → 배포 완료 후 생성된 링크 접속 → 그 계정으로 로그인.

### 설정 — 모델 / 메시징
- **Provider(LLM 모델)**: 영상은 **OpenRouter** 사용을 권장. API 키 하나로 여러 LLM(Anthropic Claude, OpenAI GPT 등)을 번갈아 쓸 수 있어, 모델별로 키를 따로 넣는 번거로움이 없다. OpenRouter에서 API 키를 발급해 입력.
- **비용 경고(중요 꿀팁)**: 초반에 고가 모델(Opus 4.7·4.6, Sonnet, DeepSeek 등)을 쓰면 비용이 폭증(발표자는 잘못 세팅해 하루 6~7만 원 소모). 이것저것 만져 보는 초반에는 **저가·무료 모델(GPT 미니 계열, Kimi 등)** 을 쓸 것.
- **메시징 채널(Telegram 연동)**: Telegram BotFather → `/newbot` → 봇 이름 지정 → **Bot Token** 획득 → 헤르메스 설정에 토큰 입력 → **allowed user ID**(토큰 문자열 앞쪽의 번호)를 입력 → Save. (토큰·유저 ID는 별도 기록해 둘 것 — 봇을 여러 개 만들면 관리가 어려워짐.)
- 연동 후 대시보드 대화창 또는 Telegram에서 `set home` 등으로 사용자(집)를 설정하면 대화가 연결된다.

### 활용 기능
- **Cron(스케줄)**: 정해진 시각에 작업 실행(예: 아침 8시 Meta 광고 리포트 전송). 대시보드 UI로 생성하거나, 에이전트에게 자연어로 "○○ 보내주는 크론 생성해줘"라고 부탁하면 알아서 만들어 등록한다.
- **Skills**: 결과물이 나올 때마다 에이전트가 스킬을 자가 생성·개선하며 사용자에게 맞춰 간다.
- **Profiles(직원 생성)**: 여러 에이전트 프로필을 만들 수 있고, 각 프로필도 `SOUL.md`·시스템 프롬프트를 가진다(특정 인물 복제 등).
- **Config**: API 실패 시 재시도 횟수, 메모리 처리 방식 등 세부 값 조정.
- **모델 동적 전환 꿀팁**: 평소엔 저가 모델로 두고, 중요한 작업(예: 양질의 뉴스레터)이 필요할 때 "모델" 명령으로 Opus 4.6 등 고성능 모델로 즉시 전환 — "어떤 수준의 머리를 빌릴지"를 작업마다 선택.
- **MCP 연동(Composio)**: **Composio**는 Notion·Google Drive·YouTube·Jira 등 여러 앱을 MCP 계정 하나로 통합 연동해 주는 플랫폼(OpenRouter가 모델을 통합하듯, Composio는 앱 연동을 통합). 두 연동 방식 중 쉬운 쪽을 쓰면 토큰을 일일이 모으지 않아도 된다.
- MD 파일(`SOUL.md`·`AGENT.md`·`USER.md`)은 직접 편집할 수도 있지만, Telegram 대화만으로도 수정·반영된다.

### 비용 · 메모리 구조
- 가장 큰 비용은 서버비가 아니라 **API(LLM) 비용**. 서버비는 상대적으로 저렴.
- 메모리(MD 파일) 용량: 통상 **5GB까지 쌓이는 데 1~2년**(많이 쓰면 더 빨라짐). 한계에 도달하면 Pro 플랜(약 $20/월, 250GB)으로 업그레이드.
- MD 파일은 휘발되지 않고 누적되므로, 향후 더 나은 에이전트가 나오면 **MD를 추출해 이식**하면 학습된 '나에 대한 자산'을 그대로 옮길 수 있다.

## 핵심 인사이트
- 헤르메스의 본질은 "결과물 → 스킬 파일화 → 피드백 반영"의 학습 루프로 **사용자 맞춤형으로 계속 깎여 가는** 에이전트라는 점. 깎을수록 가치가 누적된다.
- 통합 레이어 전략이 반복된다: 모델은 OpenRouter로, 앱 연동은 Composio로 **하나의 계정에 묶어** 관리 복잡도를 낮춘다.
- 도구 자체보다 **AI를 어디에 적용할지(적용처)와 본인 업무의 전문성·암묵지**가 성패를 가른다는 관점(발표자가 반복 강조).

## 용어 · 외부 참조
- **헤르메스(Hermes) 에이전트**: 대화·작업을 통해 자가 학습하며 사용자에게 맞춰지는 AI 에이전트.
- **셀프 임프루빙(Self-improving)**: 작업 결과물을 스킬 파일로 만들고 피드백으로 다듬는 자가 개선 학습 루프.
- **Railway**: GitHub 템플릿 원클릭 배포를 지원하는 서버 호스팅 플랫폼(Hobby ~$5, Pro ~$20).
- **OpenRouter**: API 키 하나로 여러 LLM(Claude·GPT 등)을 사용하는 모델 통합 라우팅 서비스.
- **Composio**: 여러 앱(Notion·Google Drive 등)을 MCP 계정 하나로 통합 연동하는 플랫폼.
- **Telegram BotFather / `/newbot` / Bot Token / allowed user ID**: 텔레그램 봇 생성·인증 요소.
- **Cron**: 지정 시각에 작업을 실행하는 스케줄 기능.
- **SOUL.md / USER.md / MEMORY.md / SKILL.md**: 에이전트의 성격·사용자취향·맥락·스킬을 담는 마크다운 파일.
- **VPS**: 24시간 상주를 위해 빌리는 가상 서버.
- 비교 대상으로 언급된 '오픈클로': 메신저로 호출해 PC 작업을 시키는 에이전트(자가 진화는 없음).
