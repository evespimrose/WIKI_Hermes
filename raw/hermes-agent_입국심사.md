### ☐ Atom 1: 관측 훅 ↔ 미들웨어 2면 분리 (Observe vs Mutate)

**추천도**: ⭐⭐⭐⭐⭐ 5/5
**기여도**: 워크플로우 적합도 **매우 높음**(에이전트 확장면을 "무슨 일이 일어났나=읽기전용 텔레메트리"와 "무슨 일이 일어날지=동작변경 래퍼"로 가르는 계약 — [[LLM_통제_철학_6대원칙]] "훅>프롬프트"의 형식화) · 신규성 **새 패턴**(이 볼트 훅들은 동작하나, 두 면을 *계약·스키마버전·fail-open*으로 분리 명문화한 노트 없음)
**토큰 경제학**: 턴당 비용 **+0%**(개념 노트) · 조건 일회성(설계 학습) · Benefit 높음(훅 설계 시 "관측 전용 vs 변경 가능"을 의식적으로 구분) · Risk 없음
**근거**: 기여 이유 — 관측 훅(`pre/post_api_request`·`pre/post_tool_call`)은 반환 무시·fail-open, 미들웨어(`llm/tool_request`·`llm/tool_execution`)는 페이로드 교체·`next_call` 체인. "tool_request 미들웨어가 approval *앞*에서 동작 → 재작성된 경로가 정책 대상"이라는 보안 함의까지 명세됨. 토큰 계산 0(노트화). 리스크 평가 없음
**판정 코멘트**: 단독 노트 `관측_vs_미들웨어_2면분리.md` 또는 [[LLM_통제_철학_6대원칙]] 확장. 이 볼트 PreToolUse 차단 훅·Stop 훅을 "어느 면에 속하는가"로 재분류하는 메타 기준이 된다.

### ☐ Atom 2: 캐시 보존형 컨텍스트 주입 (user-메시지 주입 + 시스템 프롬프트 불변)

**추천도**: ⭐⭐⭐⭐⭐ 5/5
**기여도**: 워크플로우 적합도 **매우 높음**(이 볼트의 캐시 안정성 강박 — cache-aligner 스킬·session-start 불변 prefix 규율 — 과 정면 대응하는 외부 실증) · 신규성 **새 실증**(원칙은 있으나 "메모리를 어디에 주입하면 캐시가 사는가"의 구체 기법 노트 없음)
**토큰 경제학**: 턴당 비용 **+0%**, 실제 규율 채택 시 **캐시 적중률↑ → 비용 절감 방향** · 조건 일회성/규율 · Benefit 높음 · Risk 없음
**근거**: 기여 이유 — honcho는 가변 메모리를 **시스템 프롬프트가 아니라 user 메시지에 API 호출 시점 주입**하고 `<memory-context>` 펜스로 "배경 데이터"임을 명시, 시스템 프롬프트엔 정적 헤더만 둔다. 다음 턴 입력에서 누출된 펜스 블록은 재처리 전 sanitize(메시지 히스토리 재유입 방지). 토큰 계산 0(노트). 리스크 없음(오히려 절감)
**판정 코멘트**: 최우선 채택. 신규 노트 `캐시_보존형_컨텍스트_주입.md` 또는 cache-aligner/[[맥락_오염_방지]] 확장. "불변 prefix·가변 suffix" 규율의 *메모리 주입* 적용 사례로 위치.

### ☐ Atom 3: observations(통합 신념) vs raw facts(원증거)

**추천도**: ⭐⭐⭐⭐ 4/5
**기여도**: 워크플로우 적합도 **높음**(주입은 중복제거·통합된 신념을, 원증거는 별도 — [[4계층_메모리_아키텍처]]·[[위키_저장_5대필터]]의 "원자적·통합 지식"과 교차) · 신규성 약간 다름(볼트는 원자노트 철학은 있으나 "통합층 주입 vs 증거층 보관"의 토큰밀도 논거는 미명문)
**토큰 경제학**: 턴당 비용 **+0%**(개념). 실제 주입 메커니즘화 시 통합신념이 원증거 다수보다 **토큰당 밀도 우위** · 조건 일회성 · Benefit 중상 · Risk 없음
**근거**: 기여 이유 — hindsight는 recall 기본을 `observation`(증거수·신선도 포함 통합 신념)만으로 좁힘. 이유: per-turn 주입엔 "1개 observation이 요약하는 raw fact 여러 개를 먹이지 않는 것"이 토큰당 밀하다. 토큰 계산 0. 리스크 없음
**판정 코멘트**: [[4계층_메모리_아키텍처]]에 "주입층=통합신념, 보관층=원증거" 1섹션 보강. 볼트 atomic-note 압축 철학의 *주입 토큰 경제학* 근거로 활용.

### ☐ Atom 4: layer-1 패턴 보안 가드 (토큰 0 로컬 정규식 → 결과 경고 append)

**추천도**: ⭐⭐⭐⭐ 4/5
**기여도**: 워크플로우 적합도 **높음**(이 볼트의 PreToolUse 차단 훅 계열에 직결되는 "토큰 0 가드" 실증) · 신규성 새 사례(볼트 훅은 차단형, 이건 *경고형 self-heal*)
**토큰 경제학**: 턴당 비용 **+0% LLM 토큰**(로컬 정규식). 경고 매치 시 도구결과 +수십 토큰/매치 · 조건 상시(경량 훅) 또는 개념 · Benefit 중상 · Risk 낮음(오탐 mediocre)
**근거**: 기여 이유 — `write_file`/`patch` 내용이 25룰(eval·pickle.load·shell=True·`torch.load(weights_only 누락)`·`dangerouslySetInnerHTML`·GitHub Actions `${{github.event.*}}` 등) 매치 시 결과에 경고 append, **파일은 쓰이고** 모델이 다음 턴 자가수정. 경로필터+lookbehind로 오탐 저감(`model.eval()` 제외). `SECURITY_GUIDANCE_BLOCK=1`로 차단 모드 전환. Anthropic 패턴 Apache-2.0 포크. 토큰 계산: LLM 0. 리스크: 오탐 → "경고일 뿐" 명시로 수용
**판정 코멘트**: [[LLM_통제_철학_6대원칙]] 또는 훅 노트에 "차단 vs 경고형 self-heal 가드" 비교 1섹션. 이 볼트 codegraph-gate/sonar-guard(차단형)와 대비되는 *비차단 자가수정* 설계축으로 기록.

### ☐ Atom 6: 다단 케이던스 변증 메모리 (cadence × depth × reasoning-level 3직교축)

**추천도**: ⭐⭐⭐ 3/5
**기여도**: 워크플로우 적합도 **중간**(base 컨텍스트=싸고 잦게 + dialectic 보충=비싼 LLM추론을 드물게, 각각 독립 케이던스로 비용통제 — 메모리 경제학 패턴) · 신규성 다름(볼트에 "메모리 갱신 빈도를 비용축으로 분리"한 노트 없음)
**토큰 경제학**: 턴당 비용 개념 **+0%**. 실제 도입 시 dialectic LLM 호출이 **추가 비용**(드물게/저-reasoning으로 상쇄 설계) · 조건 일회성(개념) · Benefit 중간 · Risk 중간(자율 LLM 추론 호출 = 통제 철학과 긴장)
**근거**: 기여 이유 — honcho의 `contextCadence`(얼마나 자주)·`dialecticDepth`(몇 패스 1–3)·`dialecticReasoningLevel`(얼마나 세게)의 3직교 노브 + cold/warm 프롬프트 자동선택 + 토큰예산 truncate. "메모리 품질을 비용 노브로 다이얼"하는 설계. 토큰 계산: 개념 0, 도입 시 dialecticCadence로 분모 키워 상쇄. 리스크: LLM이 메모리 추론까지 자율 → 결정론 약화
**판정 코멘트**: 개념만 [[4계층_메모리_아키텍처]] "메모리 갱신 비용 케이던스" 1섹션으로. 실제 변증 추론 도입은 볼트의 결정론 통제와 긴장 → 도입은 비권장, 비용통제 *축 분리* 아이디어만 차용.
