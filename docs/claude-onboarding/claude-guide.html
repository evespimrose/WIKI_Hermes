Claude 초급 가이드

1. Claude란 무엇인가?

Claude는 Anthropic에서 개발한 대형 언어 모델(LLM)로, 인간과 유사한 방식으로 텍스트를 이해하고 생성하는 AI입니다. Claude는 특히 안전성, 일관성, 복잡한 추론 능력에 초점을 맞추고 개발되었습니다.

2. LLM의 종류와 특징

2.1 주요 LLM 종류

LLM개발사주요 특징
GPT 시리즈OpenAI범용적 성능, 빠른 응답
Claude 시리즈Anthropic안전성, 긴 문맥 처리
Gemini 시리즈Google멀티모달(텍스트+이미지+동영상)
Llama 시리즈Meta오픈소스, 커스터마이징 용이

2.2 Claude의 주요 시리즈

  • Claude Opus (4.8): 최고 성능, 복잡한 추론
  • Claude Sonnet (4.6): 성능과 속도의 균형
  • Claude Haiku (4.5): 빠른 응답, 경량 작업에 적합

3. Claude의 핵심 특징

3.1 긴 문맥 처리 능력

Claude는 매우 긴 텍스트(최대 200만 토큰)를 한 번에 처리할 수 있습니다. 이는 긴 문서, 코드베이스 전체를 분석하는 데 유용합니다.

3.2 안전성과 윤리

Anthropic의 Constitutional AI 접근 방식을 사용하여 유해한 내용 생성을 최소화합니다.

3.3 구조화된 출력 지원

XML 태그를 사용해 구조화된 응답을 쉽게 생성할 수 있습니다.

4. Claude의 작업 방식: 스킬, 훅, 룰, 에이전트, 커맨드

4.1 스킬(Skill)

특정 작업을 자동화하기 위한 사전 정의된 기능 세트입니다.

예시:
  • code-review: PR이나 코드 변경점을 분석하고 피드백 제공
  • debugger: 런타임 오류 로그 수집 및 분석
  • commit-message: Git 커밋 메시지 작성 도움

4.2 훅(Hook)

특정 이벤트가 발생했을 때 자동으로 실행되는 기능입니다.

예시:
  • session-start: 새 세션이 시작될 때 자동으로 프로젝트 README 읽고 환경 확인
  • file-save: 파일이 저장될 때마다 문법 오류 검사
  • commit-pre: Git 커밋 전에 코드 스타일 자동 정리

4.3 룰(Rule)

Claude가 따라야 할 행동 규칙과 가이드라인입니다.

예시:
  • 이 프로젝트의 모든 코드 주석은 한국어로 작성하세요
  • API 키나 비밀번호는 절대 로그에 남기지 마세요
  • 복잡한 로직을 변경하기 전에 항상 사용자에게 확인하세요

4.4 에이전트(Agent)

자율적으로 작업을 수행하는 AI 에이전트로, 복잡한 작업을 단계별로 처리합니다.

예시:
  • React 프로젝트 초기 설정 에이전트: 폴더 구조 생성 → package.json 설정 → 기본 컴포넌트 작성 → 의존성 설치까지 자동으로
  • 버그 수정 에이전트: 오류 로그 분석 → 원인 추론 → 문제 정의 → 수정 코드 제안 → 테스트 실행

4.5 커맨드(Command)

사용자가 직접 실행하는 명령어로, 특정 기능을 즉시 호출합니다.

스킬과의 차이점:
구분스킬(Skill)커맨드(Command)
사용 방식자동으로 호출되거나 작업에 맞춰 사용사용자가 직접 명령어로 호출
목적복잡한 작업 자동화 (여러 단계)단일 기능 즉시 실행
예시코드 리뷰, 문서 생성컨텍스트 초기화, 세션 복원
예시:
  • /continue: 중단된 작업 계속하기
  • /resume: 이전 세션 복원하기
  • /clear: 현재 컨텍스트 초기화
  • /doc-context: 문서 파일을 컨텍스트로 로드하기

5. MCP와 플러그인

5.1 MCP (Model Context Protocol)

MCP는 Claude가 외부 도구와 서비스에 연결할 수 있는 표준 프로토콜입니다. 이를 통해 Claude는 파일 시스템, 데이터베이스, API 등과 상호작용할 수 있습니다.

5.2 플러그인(Plugin)

플러그인은 Claude의 기능을 확장하는 모듈로, 특정 작업에 특화된 기능을 추가합니다.

6. Claude의 작업 원리 요약

1. 입력 받기 → 2. 문맥 이해 → 3. 추론 → 4. 응답 생성 → 5. 결과 반환

Claude는 사용자의 요청을 이해하고, 주어진 문맥 내에서 최선의 응답을 생성합니다. 필요한 경우 외부 도구를 호출하여 더 정확한 결과를 제공할 수도 있습니다.

7. 마무리

Claude는 단순한 텍스트 생성 이상의 가치를 제공하는 AI로, 복잡한 작업을 자동화하고 효율성을 높이는 데 큰 잠재력을 가지고 있습니다.

1. 트랜스포머(Transformer) 개념

트랜스포머는 현대 LLM의 기반이 되는 신경망 구조입니다.

1.1 인코더(Encoder)

입력 데이터를 이해하고 특징을 추출하는 부분입니다.

1.2 디코더(Decoder)

추출된 특징을 바탕으로 출력을 생성하는 부분입니다.

1.3 인코더-디코더 구조

둘을 함께 사용하는 구조로, 번역 작업 등에 적합합니다. (예: T5)

2. LLM 기본 지식

2.1 LLM 동작 원리

링크: LLM은 어떻게 작동하는가? (한빛미디어)

2.2 LLM 역사와 뉴스

링크: LLMs Practical Guide (GitHub)

2.3 LLM의 환각(Hallucination)

환각이란 LLM이 사실이 아닌 정보를 마치 사실인 것처럼 생성하는 현상입니다.

대비책:
  • 사실 확인 단계 추가
  • 여러 소스 cross-check
  • "확실하지 않으면 모른다고 말해" 프롬프트로 개념 선주입

3. 모델 분류

3.1 생성형 vs 판별형

  • 생성형: 새로운 콘텐츠를 만듦 (예: GPT, Claude, DALL-E)
  • 판별형: 분류/판단만 함 (예: 이미지 분류 모델)

3.2 추론 모델 vs 비추론 모델

  • 추론 모델: 단계별로 생각하며 답변 (예: Claude with Chain-of-Thought)
  • 비추론 모델: 직접 답변 생성

3.3 대규모 모델 vs 소규모 모델

  • 대규모: 높은 성능, 높은 비용 (예: GPT-4, Claude Opus)
  • 소규모: 빠르고 저렴, 특정 작업에 최적화 (예: Llama 3 8B)

3.4 폐쇄형 vs 개방형

  • 폐쇄형: API로만 접근, 내부 공개 안 됨 (예: GPT, Claude)
  • 개방형: 가중치 공개, 자유롭게 수정 가능 (예: Llama, Mistral)

3.5 Auto-regressive vs dLLM

동작 원리 차이:
  • Auto-regressive: 이전 토큰 기반으로 다음 토큰을 순차적으로 예측 (대부분의 LLM, ex: GPT, Claude)
  • dLLM (Diffusion LLM): 노이즈를 점진적으로 제거하는 확산 프로세스로 텍스트 생성 (초기 랜덤 노이즈 → 점진적으로 의미 있는 텍스트로 변환)

3.6 언어 모델 vs 동작 모델

  • 언어 모델: 텍스트 이해/생성 (예: GPT, Claude)
  • 동작 모델: 행동 계획/실행 (예: 로봇 제어 AI)

4. 토큰(Token)이란?

토큰은 Claude가 텍스트를 처리하는 기본 단위입니다. 대략 4개의 영문자 = 1토큰, 한글 1자 = 1~2토큰 정도로 생각하시면 됩니다.

예시:
  • "Hello" → 1토큰
  • "안녕하세요" → 2~3토큰

4.1 각 Claude 모델의 스펙과 특성

기본 스펙

5. 컨텍스트(Context) & 세션(Session)

5.1 컨텍스트

현재 대화에서 Claude가 기억하고 있는 정보의 총합입니다. 대화가 길어질수록 컨텍스트도 커집니다.

5.2 세션

하나의 대화 단위를 의미합니다. 새로운 세션을 시작하면 이전 대화 내용이 초기화됩니다.

9. 서브에이전트(Sub-agent) 개념

서브에이전트는 메인 에이전트가 분할한 작업을 담당하는 보조 AI 에이전트입니다.

9.1 활용 예시

  • 사용 주체: 메인 Claude가 복잡한 프로젝트를 분석할 때
  • 메인 에이전트가 작업을 여러 하위 작업으로 분할
  • 각 서브에이전트가 특정 부분 담당 (예: 코드 분석, 문서 작성, 테스트 실행)
  • 결과를 메인 에이전트가 종합

10. 엔지니어링 개념

10.1 프롬프트 엔지니어링

AI에게 어떤 질문/명령을 내릴지 설계하는 것입니다. 명확하고 구체적인 프롬프트가 좋은 결과를 만듭니다.

10.2 컨텍스트 엔지니어링

AI에게 제공할 배경 정보를 선별하고 구조화하는 것입니다. 관련성 높은 정보만 제공하는 게 핵심입니다.

10.3 하네스 엔지니어링

AI를 특정 워크플로우에 통합하고 자동화하는 시스템을 구축하는 것입니다.

11. 오케스트레이터(Orchestrator) 개념

11.1 인간 오케스트레이터

사용자가 직접 작업을 계획하고 AI를 지휘하는 방식입니다. 복잡한 의사결정이 필요할 때 유용합니다.

11.2 에이전트 오케스트레이터 (with 서브에이전트)

AI 자체가 작업을 분배하고 서브에이전트를 관리하는 방식입니다. 반복적이고 규칙적인 작업에 적합합니다.

Constitutional AI
Anthropic이 개발한 AI 안전성 접근 방식으로:
  • 헌법(Constitution) 원칙을 기반으로 AI 행동을 규제
  • AI가 스스로 판단하고 수정하는 과정을 거침
  • 유해한 응답을 줄이고 일관된 행동을 유도
  • 투명하고 예측가능한 AI 행동을 목표로 함
XML (eXtensible Markup Language)
구조화된 데이터 표현을 위한 마크업 언어로:
  • 사용자 정의 태그로 데이터 의미를 명확히 표현
  • 계층적 구조로 복잡한 정보를 조직화
  • Claude에서 구조화된 출력을 요청할 때 활용
  • 파싱이 쉽고 다양한 시스템과 호환성 높음
Git
분산 버전 관리 시스템으로:
  • 코드 변경 이력 추적 및 관리
  • 여러 개발자 간 협업 지원
  • 브랜치를 통한 기능 개발과 병합
  • 커밋 메시지로 변경 내용 기록
