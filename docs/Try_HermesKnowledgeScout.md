# Try — 헤르메스 에이전트를 볼트의 "지식 스카우터"로

> 작성일: 2026-06-04
> 갱신: 2026-06-04 — 사용자 코멘트 반영(raw/_scout 격리 확정 · 판례 단어장 자가학습 루프 · 게이트 배치화로 Pro 토큰 보호)
> 대상 작업: 헤르메스 에이전트(자가학습·cron·다도구)를 LLM 위키의 자율 지식 수집기로 배치 — 자는시간 리서치 → distill → raw/_scout 투입 → 배치 입국심사 게이트 → 판례 환류 자가학습 → compile-wiki 파이프라인 연결 타당성
> 조사 근거: raw 대기자 [1]AI네이티브운영·[6]헤르메스 설치활용·[4]위키 비즈니스OS + 볼트 파이프라인([[LLM_Wiki_운영체계]], [[move-to-raw]])

## 1. 현황 (As-Is)

볼트 수집은 **전수동**:
```
사람이 자료 발견 → move-to-raw → raw/ → compile-wiki → obsidian
```
"무엇이 수집할 가치 있는가"의 *스카우팅을 사람이* 수행. 볼트는 사람이 먹일 때만 성장한다. [[move-to-raw]]는 입력 창구일 뿐, *발견*은 자동화돼 있지 않다.

## 2. 제안 구조 (To-Be)

헤르메스가 수집 funnel의 *앞단*을 자동화:
```
Hermes(스카우트, 서버상주·cron·자는시간 리서치)
  → 판례 단어장으로 키워드 가중 → 자가 사전필터(음수 가중 주제 자동 탈락)
  → distill(.md + frontmatter)
  → raw/_scout/  (검역 스테이징 — 수동 raw/와 물리 분리)
  → [배치] compile-wiki a 입국심사 게이트
  → 사람 승인  (권한은 끝까지 유저)
  → 판정을 판례 단어장에 환류(자가학습)
  → compile-wiki → obsidian
```
`raw/_scout/`는 헤르메스 자율 수집 전용 검역소다. 사람이 직접 넣는 [[move-to-raw]](→`raw/`)와 물리적으로 갈라, 자동/수동 출처를 섞지 않고 자율분만 별도 심사한다.

**결정적 관찰 — 이미 절반 지어져 있다**: 현재 raw/ 대기 6개는 *전부 스카우트 프론트매터*를 갖췄다:
```yaml
source_type: youtube
source_url: https://www.youtube.com/watch?v=...
captured: 2026-06-03
status: raw
```
즉 *이미 YouTube를 raw/로 스카우팅하는 흐름이 존재*한다. 헤르메스는 이를 *상시·자율화*할 뿐이고, 방금 운용한 `compile-wiki a`(입국심사)가 *그 산출물의 품질 게이트*로 이미 작동한다.

## 2-1. 판례 단어장 (Scout Lexicon) — 자가학습 루프 [핵심]

스카우트에 *방향타*를 주는 두 산출물:

1. **볼트 지향 mandate** (정적 북극성): 이 볼트가 "무엇에 관한 곳인가". 허브 노트([[LLM_워크플로우_생태계]]·[[LLM_Wiki_지식자산_현황]])와 태그 분포에서 추출. `compile-wiki a`가 매번 첫 줄에 출력하는 "지향 방향성(추론)" 문장이 그 씨앗이다.

2. **판례 단어장** (동적·누적 사례집): 매 입국심사의 *사람 판정*을 1줄 또는 1~2단어로 정제해 누적. 각 판례가 키워드 가중치(양/음)로 환산된다:
```
✅채용  하네스 엔지니어링       → +가중: harness, orchestrator, agent team, 검증루프
✅채용  카파시 위키 패턴        → +가중: context-share, 맥락오염, 저장필터
⛔반려  NotebookLM 챗봇 레시피   → -가중: notebooklm, 지식봇제작, 활용레시피
⛔반려  n8n 도구 비교          → -가중: n8n, 노코드 비교, 도구선택
```

**자가학습 루프**:
```
Hermes 스카우트 (판례 단어장으로 키워드 가중)
  → 자가 사전필터 (음수 가중 주제 스스로 탈락 → 게이트 도달량 감소)
  → raw/_scout/ (사전 통과분만)
  → [배치] 사용자 compile-wiki a (사람 게이트·권한 유지)
  → 판정 → 1줄/1~2단어 정제 → 판례 단어장에 append
  → 다음 라운드 더 정확
```

이 루프가 영상 [6] 헤르메스의 *셀프 임프루빙*(결과→SKILL.md→피드백)을 *스카우팅 방향*에 적용한 것이다. **사람의 입국심사 판정이 곧 스카우트의 교사 신호(teacher signal)** — 사용자가 단순 수집기가 아니라 헤르메스를 택한 본래 의도가 여기에 있다. 판례가 쌓일수록 스카우트는 볼트 지향에 수렴하고, 동시에 §3-B의 게이트 토큰 부담을 *사전필터로 자가 경감*한다(코멘트 1·3이 코멘트 2로 함께 해소되는 지점).

## 3. 장단점 비교

### 3-A. 아키텍처 정합성
✅✅ **거의 완벽**. 모든 조각이 이미 존재한다 — raw 프론트매터 컨벤션, 입국심사 게이트, compile 파이프라인. 헤르메스는 빈 *발견 자동화* 슬롯만 채운다. [[move-to-raw]]의 자율 버전.

### 3-B. 런타임·비용
⚠ 두 비용원:
- **헤르메스 API**([6]: 잘못 세팅 시 일 6~7만원) — *저가모델 스카우팅 + 고가모델 distill* 분리(영상 [6] 모델 동적 전환)로 완화. 서버비(Railway ~$5/월)는 부차.
- **입국심사 게이트 토큰**(코멘트 3): `compile-wiki a`는 raw 전체+obsidian 인덱스를 통독하는 *고비용* 작업이라, 헤르메스 드롭마다 난사하면 Pro 플랜의 낮은 토큰을 빠르게 소진한다. → 게이트를 *파일당이 아니라 배치*로 돌리고, §2-1 판례 단어장 자가 사전필터로 *게이트 도달 물량 자체*를 줄인다. 사람 권한은 유지하되 호출 빈도를 통제하는 구조.

### 3-C. 현 파이프라인 연관
✅ raw 프론트매터·입국심사·compile 3단이 *그대로 수용*. 신규 배관 거의 불필요 — 헤르메스 출력 포맷만 현 raw 컨벤션에 맞추면 됨.

### 3-D. 단기 도입비용
중. ①헤르메스 배포([6] Railway 원클릭), ②볼트 지향([[LLM_Wiki_지식자산_현황]]·허브)을 스카우팅 mandate로 주입, ③raw/ 쓰기 접근(git clone or 동기화). 코드 변경보다 *설정·권한* 비용.

### 3-E. 장기 리스크
⚠ **맥락 오염**(영상 [4]) — 자율 수집이 off-axis 노이즈를 쏟는다. **증거**: 방금 입국심사가 [2]NotebookLM레시피·[3]n8n비교를 ⛔HOLD 처리. 게이트 없는 자율 수집이면 볼트가 마케팅 잡음으로 오염됐을 것. 입국심사가 *필수 안전장치*임이 실증됨.

## 4. 방향·오염 위험 분석

| 위험 | 내용 | 완화 |
|---|---|---|
| 맥락 오염 | 저품질·off-axis 자율 유입 | 입국심사 게이트 의무([2][3] HOLD가 실증) |
| 볼트 직접 쓰기 | 헤르메스가 obsidian 오염 | `raw/_scout/` 검역만 쓰기, obsidian 금지 |
| 방향 드리프트 | 스카우트가 엉뚱한 주제 수집 | 볼트 지향 mandate + 판례 단어장 가중(§2-1) + [4] 5대필터 |
| 볼륨 폭주 | raw/_scout/ 범람 | 레이트리밋 + distill(dump 금지) + 자가 사전필터 |
| 게이트 토큰 난사 | compile-wiki a 빈발로 Pro 토큰 소진 | 배치 게이트 + 판례 단어장 사전필터로 도달량 감축(§2-1) |

핵심은 [[Cave_Man_Protocol]]의 정신과 동형 — *무차별 수집 금지, 정제된 좌표만*. 스카우터는 "원시인 수집"을 해선 안 된다.

## 5. 종합 평가 및 추천

**🟡 고가치 조건부 추천**. 볼트가 *이미 80% 준비*돼 ROI 최고. 단 게이트 없이는 오염 폭탄.

**5대 조건** (코멘트 반영):
1. **스테이징 격리(확정)**: 헤르메스는 `raw/_scout/`에만 쓰기. 수동 [[move-to-raw]](→`raw/`)와 물리 분리. obsidian 직접 금지.
2. **인간 권한 유지 + 게이트 배치화**: 입국심사·승인은 끝까지 사용자. 단 드롭마다 난사 금지 — *배치*로 호출해 Pro 토큰 보호.
3. **판례 단어장 자가학습(§2-1)**: 매 입국심사 판정을 1줄/1~2단어 판례로 누적 → 키워드 가중치로 헤르메스 스카우팅 방향을 자가 보정. *본 통합의 핵심 의도*.
4. **방향성·5대필터 주입**: 볼트 지향 mandate + [4] 저장 5대필터를 스카우팅 1차 기준으로.
5. **distill, not dump**: 레이트리밋 + 원자 요약(현 raw 6개 정제도 유지) + 자가 사전필터.

**단계별 추천 순서**:
```
1. 입국심사 승인분 [4]위키패턴 먼저 컴파일 → 저장 5대필터·맥락오염 개념 확보
2. 볼트 지향 mandate 문서 작성 + 판례 단어장 빈 골격 생성(docs/ 또는 신규 schema 레이어)
3. 헤르메스 배포 + 저가모델 스카우팅 + raw/_scout/ 쓰기 권한
4. mandate + 판례 단어장을 헤르메스 스카우팅 기준으로 주입
5. 1주 1주제 파일럿 → raw/_scout/ 배치 적재
6. 배치 compile-wiki a → 판정을 판례 단어장에 환류 → 가중치 정확도(승인/보류 비율) 측정 → 신뢰 시 상시화
```

## 6. 미결 사항 연계

- **Q3과 결합**: 헤르메스의 볼트 *읽기*(공유 두뇌, Q3)와 *쓰기*(스카우터, 본 try)는 한 쌍. 읽기는 즉시 가능, 쓰기는 본 게이트로 제어.
- **입국심사 의존**: 본 스카우터의 품질 기준인 [4]저장 5대필터·맥락오염([[맥락_오염_방지]] 미래 노트)은 raw/ 대기 중. try 우선 처리 후 컴파일 예정.
- **판례 단어장 위치·포맷 미정**: `docs/scout_lexicon.md`(누적 사례집) + 볼트 지향 mandate를 어디 둘지(docs/ vs 신규 `schema/` 레이어) 결정 필요. 영상 [4]의 Schema 레이어(INDEX/LOG) 도입과 묶어 검토 — Q2의 "빠진 1마일" Schema 강화와 동일 과제.
- **자가학습이 본 의도**: 사용자가 헤르메스를 택한 핵심 이유는 단순 수집이 아니라 *입국심사 판정으로 훈련되는 방향 자가학습*(§2-1). 판례 누적이 스카우트를 점점 볼트 지향에 수렴시키고, 게이트 토큰 부담을 사전필터로 자가 경감한다.
- **수집 흐름 정식화**: 성공 시 [[LLM_Wiki_운영체계]]의 3단계 파이프라인에 "0단계: 자율 스카우팅 + 판례 환류"가 선행 추가되는 구조 진화.
