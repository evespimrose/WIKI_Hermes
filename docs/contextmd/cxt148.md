## Background
These are roadmap-generated markdown docs intended for on-demand RAG, not main context. Goals:
- Preserve context via compression
- Create govern/concept nodes
- O(1) pop when needed
- Keep main context window KV cache keyword-excluded blank while maintaining input/output persistence with 0 hallucination
- On-demand RAG with codegraph nodes linked to BLK-based codebase + inter-doc backlinks

## 4 Adversarial Vulnerabilities in On-Demand RAG Nodeization

### 1. Dark Node Phenomenon Due to Missing Meta-BLK
- **Attack Path**: Domain code has clear BLK indexes, but meta-policy/infrastructure docs don't fit specific BLK categories
- **Vulnerability**: With blank main context, if these docs don't strictly use abstract layer indexes like BLK-SYS/BLK-POLICY, RAG engine can't find them → nodes exist but never get popped (dark nodes)
- **Result**: AI hallucinates by following old policies

### 2. Semantic Trigger Loss from Compaction (Trigger Inversion)
- **Attack Path**: Extreme compaction removes natural language semantic surface for RAG keyword triggering
- **Vulnerability**: Garbage context words (stale, drift, parity pass, byte-identical) removed, cosine similarity drops between sparse main context keywords and compacted nodes
- **Result**: RAG nodes miss search filters despite being perfect matches

### 3. Stale Node Drift on Policy Updates
- **Attack Path**: dict-sync-check.sh only triggers on .cs/.ts changes, no gate for re-compacting codegraph nodes when infra/scripts change
- **Vulnerability**: Manual tweaks to .trae/rules/validate-cxt.md don't update fixed codegraph nodes
- **Result**: RAG pops stale nodes → collision with local filesystem → paradox where hallucination-0 nodes cause hallucinations

### 4. Context Window Explosion from Backlink Cascade
- **Attack Path**: Backlinks follow when popping nodes
- **Vulnerability**: Allowing all link node references mechanically causes dozens of related nodes to inject into main context at once when popping one policy node
- **Result**: Violates token efficiency principle, explodes KV cache in one turn

### How Schema Fixes 4 Vulnerabilities
1. **meta_blk and triggers**: Embeds search-engine-specific semantic lexicon in header to prevent RAG loss from natural language removal
2. **source_checksum**: Records target file hashes for hooks to immediately detect and invalidate stale nodes
3. **Backlink visibility control**: Only backlink list text kept in header; blocks linked node body cascading load until explicit tool call

All work must deadly and strictly follow the rules defined in SKILL.md and CLAUDE.md.
