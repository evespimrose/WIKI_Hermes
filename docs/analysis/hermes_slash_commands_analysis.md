# Report — Hermes 슬래시 명령어 튜토리얼 분석

> 생성일: 2026-06-20
> 대상: https://www.youtube.com/watch?v=vfhXPrMp15A — "Hermes를 실무 에이전트로 바꾸는 명령어들" (채널: 일하는 ai, 게시: 2026-05-11)
> 조사 범위: 한국어 자동 자막 전문 ✅ (7,105자 / 1,873단어). 영상 화면·데모 산출물은 자막에 묘사된 범위만. 구현 코드·문서 미접근.
> **출처 한계(정직성)**: 본 분석의 유일 근거는 **자막 전사문**이다. 발표자가 화면에서 시연한 결과를 말로 묘사한 것이며, 실제 동작·구현은 코드로 검증하지 못한다. 따라서 §3은 *코드 대조*가 아니라 **표방(영상 제목·서사) 대 자막이 실제 보여준 것 + 볼트 기존 Hermes 자료와의 중복**으로 수행한다. 자막 TTS 오인식(예: "리튬"=/resume 추정, "오픈 클로우"=OpenClaw 추정, "LM 위키"=LLM 위키)은 추정으로 명시한다.

## 1. 무엇을 위해 개발되었는가 (목적)

한 줄 정의: **이미 설치한 Hermes 에이전트를 "날씨나 묻는" 수준에서 벗어나 실무에 쓰기 위한 슬래시 명령어 사용법 입문 튜토리얼.** 대상은 Hermes(또는 동류 CLI/모바일 에이전트)를 막 설치한 비개발자~입문 사용자다.

영상이 표방하는 가치는 "명령어를 알면 같은 에이전트를 훨씬 잘 부린다"이다. 발표자는 본인이 Codex와 Hermes를 병용하며, Hermes가 "워낙 자동화돼 있고 자동 업데이트되고, 말만 하면 다 실행·연결해 줘서" OpenClaw(자막 "오픈 클로우" — 추정)를 안 쓰게 됐다고 증언한다. 즉 영상의 실제 목적은 **제품 온보딩 보조** — 슬래시 명령 UI를 하나씩 짚어주는 데모다.

## 2. 어떻게 달성했는가 (구현)

영상은 "구현"이 아니라 **명령어 시연 나열**로 달성한다. 자막이 드러낸 명령 흐름(원문 표기 → 추정 정규형):

- **세션 점검**: `스테이터스`(/status) — 세션 ID·제목·생성시각 표시(모바일도 동일). `푸터`(/footer) — 모델명·토큰 사용률(예: "GPT 5.4, 11%") 표시 토글. `헬프`(/help) — 주요 명령 목록.
- **모델 제어**: `리즈닝`(/reasoning) — 사고 단계 low/medium/high/x-high(자막 "맥스"는 정정해 "X하이") 전환. 모델 자체 교체(코덱스 5.4-mini → 5.5)로 "더 똑똑한 답변".
- **컨텍스트 관리**: `컴프레스`(/compress) — 토큰/컨텍스트가 과하게 찼을 때 현재 대화를 요약·압축해 효율화. `로`(/yolo 추정) — 매 단계 승인을 생략하는 자동승인 모드.
- **리서치·위키**: 링크(유튜브 등)를 그냥 던지면 web-extract·browser-navigation·browser-click 툴을 **자동 선택·호출**. arXiv 최근 논문 조사 → 위키에 PDF 저장+요약+개념 노트로 Obsidian 통합 시연. `옵시디안`(/obsidian) — LLM 위키 볼트 경로를 찾아 검색.
- **세션 항해**: `리튬`(/resume 추정) — 과거 대화 세션 목록·재개. 세션 포인트(자막 "우물자"=`#` 체크포인트)를 기준으로 `롤백`(/rollback 1·2…)로 되감기. `브랜치`(/branch) — 토픽은 유지한 채 분기해 케이스 A/B/C 실험, /resume로 원본 복귀.
- **실행 개입 3종**: `Q`(/q, "약속" — 다음 턴에 요약 후 실행), `백그라운드`(/background, "지금 당장"), `스티어`(/steer, "방향 전환" — 다음 단계에 지시를 끼워 재집중).
- **장기 목표**: `골`(/goal) — 장기 목표 설정 + 턴 버짓(예 20턴) + "성과로 나타날 때까지" 자율 지속.
- **운용**: `워크스페이스`(/workspace, 구글 API로 캘린더·이메일 확인 — 자격증명 필요), `디버그`(/debug, 로그+시스템정보 리포트 업로드), `업데이트`(/update, 최신 버전 자동 풀), `리스타트`(/restart).

## 3. "정말로" 그러한가 (적대적 검증)

자막 한계 안에서 **영상의 서사 ↔ 자막이 실제 보여준 것 ↔ 볼트 기존 자료**를 대조한다.

**표방대로 보여준 것:**
- 슬래시 명령들이 실제로 존재하며 모바일에서도 동작함은 시연으로 확인 ✅
- 링크만 던져도 적합한 툴(web-extract/browser)을 **자동 선택**한다는 점은 데모로 보여줌 ✅
- "LM 위키" 기능으로 arXiv 논문이 PDF+개념 노트로 Obsidian에 들어오는 결과 화면 제시 ✅

**표방하나 실제는 다른/약한 것:**

| 주장 | 실제(자막 기준) |
|---|---|
| 제목: "**실무 에이전트로 바꾸는** 명령어들" | 시연된 명령 대다수(`/status`·`/footer`·`/help`·`/update`·`/restart`·`/debug`)는 **모든 현대 CLI 에이전트에 공통인 범용 UX 노브**다. "실무 전환"의 지렛대는 명령 목록 자체가 아니라, 그 안에 묻힌 소수 개념(인터럽트 3종·자가확장 위키)이다 |
| "LM 위키만 세팅하면 지식이 알아서 팽창" | 발표자 스스로 부정 — 기본 세팅으로는 "로우/옵션이 제대로 안 들어오고", 노트들이 "단독(고립)"으로 남으며 "아직 최적화 못 한 상태"라 인정. **자율 확장은 사람이 계속 멘트로 지시해야** 일어난다(set-and-forget 아님) |
| (암묵) 새로운 에이전트 능력 소개 | `/goal`은 발표자 본인이 "코덱스에 먼저 들어왔는데 헤르메스가 따라 구현"했다고 증언 — **크로스 에이전트 모방**의 결과이지 Hermes 고유 신규성 아님 |
| (암묵) 처음 보는 정보 | 본 볼트는 이미 Hermes 자료 2건 보유 — GitHub 아키텍처 해부(`hermes-agent_analysis.md`)와 선행 유튜브 raw(`헤르메스_에이전트_자가학습_설치활용__yt_wiVd1CwfBls.md`). 본 영상의 운용 how-to는 그 "설치·활용" 영역과 **상당 부분 재중복** |

**숨은 전제 / 결론:** 이 영상의 이전 가능 가치는 "명령어 카탈로그"가 아니라 그 안에 비명시적으로 들어 있는 **세 개의 에이전트 제어 개념**이다 — (1) 실행 중 에이전트에 개입하는 **인터럽트 3종 분류**(예약/즉시/방향전환), (2) 대화를 버전 트리로 다루는 **롤백 vs 브랜치**, (3) 위키를 *지속 지시로* 자가확장시키는 구성 철학. 나머지(상태·푸터·헬프·업데이트·워크스페이스·디버그·모델교체)는 휘발성 제품 UI로, `vault_mandate`의 out-of-scope("설치 튜토리얼·활용 레시피")와 `scout_lexicon` 판례 [6]("헤르메스 설치·활용 🟡조건부, −설치튜토리얼")에 정면으로 걸린다. **가치는 명령의 표면이 아니라, 명령이 구현한 *제어 추상*에서만 나온다.**

## 4. LLM 위키 워크플로우에 무엇을 기여할 수 있는가

`schema/vault_mandate.md`(북극성: "LLM/AI 에이전트를 결정론적으로 통제하는 워크플로우 엔지니어링 + 지식 누적 위키 메타시스템") 및 `scout_lexicon.md` 판례 기준.

**In-scope 교차점:**
- **인터럽트 3종(예약/즉시/방향전환)** ↔ [[하네스_엔지니어링]] · [[3Tier_에이전트_스튜디오]] — "실행 중 에이전트를 어떻게 제어하는가"의 구체적 프리미티브. 기존 노트에 명시 대응 없음 → **신규 개념 후보**.
- **위키 자가확장 구성 철학**(백링크 강제·부분 URL 수용·무지시 자율 확장·태그=키워드+시그널 통제코드·고립노트 연결 분석) ↔ [[카파시_LLM_위키_패턴]] · [[위키_저장_5대필터]] · [[Raw_Wiki_Schema_3레이어]]. 동시에 `vault_mandate` **갭 Priority 3 "웹 링크 지식 정제 자동화(Hermes)"**([[Try_HermesKnowledgeScout]] 타당성만 존재)의 **실사용 증언**. 단, 정적 카파시 패턴과 부분 중복 → 확장 위치.
- **/goal 턴 버짓 + 크로스 에이전트 전파** ↔ `vault_mandate` 갭 Priority 2 "/goal+/loop 통합"(Mastermind) · "강제 컴팩션 트리거 Hook". 신규 개념은 아니나 **로드맵 가설을 뒷받침하는 외부 실증**.
- **롤백 vs 브랜치(대화=버전 트리)** ↔ [[4계층_메모리_아키텍처]] · [[컴팩션_생존_전략]] · [[RIPER_5단계_상태머신]] — 대화 상태를 되감기/분기하는 UX. 단, 본 볼트의 *결정론적 훅* 철학과는 결이 다른 *대화형 UX* → 긴장 존재.

**구체적 기여 가능 개념:**
1. **인터럽트 3종 분류** — 예약(다음 턴 약속) / 백그라운드(즉시 병렬) / 스티어(현재 실행 방향 재지정). 에이전트 오케스트레이션의 미세 제어 어휘.
2. **위키 "지속 지시형" 자가확장** — 1회 세팅으로 끝내지 말고, 매 입력마다 "연결하라·확장하라"를 지시해 고립 노트를 능동 해소하는 운영 규율. "연결된 지식만이 LLM 검색에서 의미를 가진다"는 명제.
3. **턴 버짓형 장기목표** — 목표 + N턴 예산 + "성과로 나타날 때까지" 자율 지속. /goal+/loop 갭의 구체 형상.

**불일치점(out-of-scope 인접):**
- `/status`·`/footer`·`/help`·`/workspace`(구글캘린더)·`/debug`·`/update`·`/restart`·모델교체·`/reasoning`·`/yolo` → 제품 운용 how-to. `vault_mandate` "활용 레시피" 경계 + `scout_lexicon` −활용레시피/−설치튜토리얼.
- 설치·온보딩 전제 → **선행 Hermes 유튜브 raw가 이미 덮음** → 신규 노트 금지, 중복.
- `/compress` → 개념적으로 [[컴팩션_생존_전략]]에 이미 내재. 게다가 *수동* 압축이라 갭 Priority 1 "압축 키워드 *자동* 추출 규약"을 채우지도 못함.

**기여 판정: 개념 추출 조건부.** 원자 2~3개(인터럽트 3종·자가확장 규율·턴버짓 목표)만 *기존 노트 확장*으로 흡수, 나머지 명령 카탈로그는 보류.

## 5. 핵심 로직 100% 적용 시 턴당 토큰 증가 추정

본 출처는 *런타임 시스템*이 아니라 *지식 원자*다. 따라서 "턴당 토큰 증가"의 정확한 답은 **채택 형태에 달려 있다**:

| 채택 형태 | 상시성 | 턴당 비용 | 근거 |
|---|---|---|---|
| 개념 원자를 obsidian 노트로 흡수 | 온디맨드 | **+0%** | [[카파시_LLM_위키_패턴]]의 온디맨드 참조 모델 — 매 턴 주입 아님. 질의 시에만 로드 |
| 인터럽트/롤백/브랜치 = 사용자 조작 | 사용자 발화 시 | +0% (시스템 비용 없음) | 사람이 치는 명령. 컨텍스트 상주 0 |
| "자가확장 위키"를 *자동 루프*로 구현 | 백그라운드 잡 | 턴당 0, **잡당 N턴** | 매 턴 주입이 아니라 별도 백그라운드 실행. 비용은 컨텍스트 세금이 아니라 *실행 턴 수*(= /goal 턴 버짓이 통제하려는 바로 그 변수) |

**베이스(가정)**: 평시 1턴 ≈ 시스템 프롬프트+훅 주입(고정) + 대화. 본 출처 원자들은 그 고정 prefix를 **건드리지 않는다**(캐시 안정성 무영향).

**결론**: 지식 원자 채택의 런타임 세금은 **+0%**(온디맨드 모델 덕). 유일한 비용은 자가확장을 *자동화*할 때의 **실행 턴 수**이며, 이는 컨텍스트 오염이 아니라 [[Producer_QualitySentinel_Reporter_게이트]]·턴버짓으로 통제할 *오케스트레이션 비용*이다.

## 종합

이 영상은 표면적으로 "Hermes 명령어 22종 입문"이지만, 정제하면 **세 개의 이전 가능한 에이전트 제어 개념**(인터럽트 3종·지속지시형 자가확장 위키·턴버짓 목표)만 남고 나머지는 휘발성 제품 UI(out-of-scope 활용 레시피)다. 게다가 운용 영역은 볼트의 기존 Hermes 자료 2건과 재중복된다. 따라서 **전체 채택이 아니라 개념 추출 조건부** — 원자 2~3개를 [[하네스_엔지니어링]]·[[카파시_LLM_위키_패턴]]·로드맵 갭 노트의 *확장*으로 흡수하고, 명령 카탈로그·온보딩·운용 how-to는 `scout_lexicon` 판례 [6]을 따라 버린다. 채택 조건: 신규 노트를 만들지 말 것(중복), 인터럽트 3종만 독립 가치가 있어 소형 신규 노트 후보로 검토 가능.
