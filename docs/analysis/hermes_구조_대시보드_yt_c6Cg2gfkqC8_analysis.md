## 1. 무엇을 위해 개발되었는가 (목적)

영상 자체는 도구가 아니라 **튜토리얼 콘텐츠**다. 한 줄 정의: **Hermes 에이전트 설치 후속편 — 내부 디렉터리 구조(`~/.hermes`)와 대시보드를 옵시디안으로 한눈에 파악하는 방법을 안내하는 워크스루.** 첫 문장("지난 시간에는 헤르메스 에이전트를 설치하고 기본까지 살펴봤습니다")이 밝히듯 이 영상은 **설치 편의 다음 회차**다. 타깃은 Hermes를 막 설치한 개인 사용자이며, 표방하는 가치는 영상 제목 그대로 *"반드시 알아야 할 구조의 파악"* + *"대시보드를 통해 스킬·칸반 등 활용"*이다.

## 2. 어떻게 달성했는가 (구현 — 영상이 보여주는 흐름)

영상이 순서대로 안내하는 경로:

- **공식 문서 활용**: 노스리서치 공식 다큐먼트(영/한). 한글로 전체 흐름을 먼저 훑고, 막히면 사람한테 묻기보다 **로그를 GPT에 주고 원인 추적**, 문서 정독이 부담이면 **Hermes에 요약을 위임**.
- **내부 구조 열람**: `cd ~/.hermes` → (맥OS) Cmd+Shift+. 로 숨김 폴더 표시 → **이 경로를 옵시디안 보관함으로 지정**해 마크다운 기반으로 내부 문서를 훑는다. 첫 인덱싱은 다소 느리나 한 번 잡으면 역할·연결 관계를 빠르게 파악.
- **default/SOUL 파일**: 에이전트의 **페르소나·답변 품질 기준·한국어 윤문 기준**. "AI가 만든 티가 너무 나지 않도록" 통과 품질을 정해 두는 곳.
- **구성 파일**: hooks·kanban·plans 등. 폴더별로 "어떤 기능을 어떻게 실행하는지"의 구조를 드러냄.
- **skills 폴더**: 각 스킬 정의. 슬래시 명령으로 호출하면 그 절차대로 실행. 각 스킬에 SKILL.md(역할·사용 시점·실행 순서)가 들어 있어 옵시디안에서 스킬 체계를 한눈에 봄.
- **대시보드**(`hermes dashboard`): 현재 세션·사용 모델·크론 자동화 상태 확인. 어느 에이전트가 어떤 스케줄로 실행했는지, 돌고 있는 크론잡과 실행 시점 표시. **스킬 on/off 토글**도 가능.
- **플러그인/외부 연결**: GitHub·browser-use·Google Meet·이미지 생성·OpenAI Codex 등. 연결 가능 여부 확인 후 **본인 작업 흐름에 맞는 것만 선택**.
- **프로파일**: default 외 별도 에이전트(예: Analyst) 관리. soul·검사 규칙·윤문 기준·plans는 물론 **샌드박스·사용 스킬·아웃풋/인풋 경로를 프로파일 기준으로 제어** — "입력은 어디서 받고 결과는 어디 저장할지"를 명확히 하면 에이전트가 자기 범위를 더 잘 이해.
- **칸반**: 할 일/하지 말 것을 정리해 Ready로 옮김. Blocked→Ready→Progress→Done. **To-Do가 곧바로 Progress로 가지 않고 "실행 조건이 맞아야" 진행 상태로 이동**(조건 게이팅). 에이전트·프로젝트 기준으로 업무가 묶이는 것도 칸반 화면에서 확인.
- **사용 지표**(environment 영역): unblock·uncovered·factor·tier 류의 사용 지표 + 토큰 비율·시스템 콜 등(자막 뭉개짐, 지표명 일부 불확실). 자주 쓰는 기능 vs 아직 안 켠 기능을 점검해 필요한 플러그인·스킬 선택에 활용.
- **셀프 이볼루션(self-evolution)**: 영상이 "Hermes가 최근 뜨는 이유"로 지목. GitHub를 설치하면 **스킬을 자체 평가한 뒤 점수가 개선될 때까지 반복 평가·개선**한다는 것. "현재는 페이즈 단계라 스킬 개선에 중점", 향후 **도구·시스템 프롬프트 구현까지 확장 예정**. 핵심 메시지: 작업 실행에서 끝나지 않고 *작업 방식 자체*를 개선.
- **레포 분석 의뢰 흐름**: 링크/GitHub 주소를 Hermes에 주면 레포+공식문서를 함께 분석해 설치·활용을 정리. "적용 상태 확인"을 시키면 설치된 기능·검색 중인 스킬·(스킬 검색·본문 분리) 데이터셋 베이스라인 평가 진행도까지 정리. 평가 후 피드백을 주면 **스킬 개선 후보를 생성**(기존본 vs 개선본 비교, 타임스탬프 붙은 아웃풋).
- **자동 실행 주의(운영 안전)**: "자동으로 실행해줘"식 위임은 의도치 않은 방향으로 갈 수 있음. 안전책 — **먼저 몇 가지 실행안으로 제한 → 장단점 비교 → 사용자가 고른 안을 기준으로 세팅**. "바로 실행을 맡기기보다 계획을 먼저 세우게 하고 사람이 선택하는 구조가 훨씬 안정적."

## 3. "정말로" 그러한가 (적대적 검증)

자막 1종이라는 한계 안에서, 영상이 **표방하는 것**과 자막이 실제로 **전달하는 것**을 대조한다.

**표방대로(영상이 실제로 보여준 것):**
- `~/.hermes`를 옵시디안 보관함으로 잡아 마크다운 내부 구조를 훑는 워크플로우 ✅ (시연 서술)
- 대시보드가 세션·크론·스킬 토글·사용 지표를 노출 ✅ (시연 서술)
- SOUL/프로파일/스킬/칸반이 별도 파일·폴더로 존재 ✅

**표방하나 실제는 다르거나 근거가 약한 것:**

| 주장 | 실제(자막 기준) |
|---|---|
| "**반드시 알아야 할 구조의 파악**" | 내용 대부분이 **대시보드 버튼 위치·토글 방법·메뉴 투어** = 휘발성 UI how-to다. "구조"라기보다 "UI 내비게이션". *왜* 그렇게 설계됐는지(관측↔미들웨어 분리, 프로바이더 레지스트리 등 repo가 드러낸 설계 골격)는 영상에 전혀 없음 |
| "셀프 이볼루션이 Hermes가 뜨는 이유" | 영상은 자가진화를 **무비판적 마케팅 그대로** 전달. repo 분석(`hermes-agent_analysis.md` §3)은 이 "self-improving"이 실은 **외부 메모리 SaaS + 스킬 파일화의 통합**이지 신규 학습 알고리즘이 아님(OpenClaw 포크 계보)을 문서로 논파했다. 영상은 그 *이미 논파된 서사*를 재생산 |
| "스킬을 자가평가해 점수 개선될 때까지 반복" | 영상 스스로 "현재 페이즈 단계", "이거 보기만 해도 살짝 어렵죠"라며 메커니즘 불투명을 인정. **평가 기준·점수 정의·수렴 보장**은 영상에 없음 |
| "사람이 읽고 직접 판단할 시간을 줄이고 필요한 부분만 뽑아 쓰는 능력이 중요" (문서 요약을 에이전트에 위임) | 실무 팁으로는 유효하나, 본 볼트의 통제 철학(사람이 최종 판단·검증)과 **긴장**한다. 단, 영상 후반의 "자동실행 금지·계획우선·사람선택"이 이 긴장을 스스로 상쇄(§4 Atom 1) |

**숨은 전제 / 결론:** 이 영상의 실제 가치는 "구조 파악"이 아니라 단 두 개의 **운영 nugget** — (a) *에이전트의 자기 마크다운 내부를 옵시디안 보관함으로 열어 본다*는 메타 기법, (b) *자동 실행 금지·계획 우선·사람이 선택*이라는 운영 안전 원칙 — 에 있다. 나머지(대시보드 투어·플러그인 토글·칸반 UI 조작·사용 지표 읽기)는 제품 사용 레시피이고, 자가진화 서사는 repo 분석이 이미 논파한 마케팅의 재생산이다. **"구조"라는 제목이 대시보드 워크스루를 과대 포장**한다.

## 5. 핵심 로직 100% 적용 시 턴당 토큰 증가 추정

**상시 적용 여부:** 영상은 도구가 아니라 운영 권고이므로 채택 형태는 전부 *개념/원칙 노트*다. 런타임 **+0%**.

| 원자 | 채택 형태 | 턴당 토큰 영향(베이스 20K 가정) |
|---|---|---|
| 자동실행 금지·계획우선·사람선택 | 원칙 노트에 외부 사례 1줄 | +0% (이미 [[RIPER_5단계_상태머신]]이 동형) |
| 옵시디안 자기 내부 열람 메타 | 개념 사례 1줄 | +0% (codegraph로 이미 보유) |
| 셀프 이볼루션 | repo 분석에 합류(confirmatory) | +0% |

**결론:** `+0%`. 신규 런타임 비용 없음. 단, 영상의 "사람 판단 시간을 줄이려 요약을 에이전트에 위임"을 *그대로* 도입하면 토큰이 아니라 **정확성(검증 누락) 비용**이 발생하므로 본 볼트는 반대 방향(사람 최종 판단)을 유지한다. 가치는 전적으로 *기존 통제 원칙의 외부 수렴 증거*에 한정된다.

## 종합

"반드시 알아야 할 구조의 파악"을 표방하지만, 실체는 대시보드/옵시디안 UI 워크스루 + 자가진화 마케팅의 재생산이다. repo 분석과 이전 편 raw가 이미 아키텍처·설치·자가학습을 덮고 있어 **3자 주제 중첩**이 크다. 정제 후 남는 금은 사실상 **단 하나의 강한 원자** — "자동 실행 금지·계획을 먼저·사람이 선택"이라는 운영 안전 원칙으로, 본 볼트의 RIPER·게이트 철학에 외부 실무자가 독립 수렴한 증거다. 그 밖(옵시디안 자기 열람 메타, 셀프 이볼루션)은 기존 노트에 1줄 외부 사례로 첨부할 수준이며 신규 노트는 불필요하다. compile 단계에서 repo 분석·이전 편 raw와 **3자 정합**이 필수다.
