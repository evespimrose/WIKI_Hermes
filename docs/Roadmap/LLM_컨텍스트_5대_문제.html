LLM 컨텍스트 5대 문제

0. 개요

장기 LLM 운용에서 반복적으로 관찰되는 다섯 가지 본질적 문제. 모든 설계 결정은 이 다섯 문제 중 하나 이상을 직접 겨냥한다.

1. 컴팩션 절벽 (Compaction Cliff)

장기 세션에서 컨텍스트가 한계에 도달하면 `/compact` 또는 자동 압축이 실행된다. 이때 현재 진행 단계, 사용자가 강조한 규칙, 진행 중 RIPER 모드, 직전 결정의 근거가 모두 소실된다. AI는 이전 컨텍스트가 무엇이었는지 모르는 채 작업을 이어가며, 사용자는 매번 처음부터 다시 설명한다.

대응 전략

  • 컴팩션 생존 전략의 L2·L3 계층
  • 4계층 메모리 아키텍처

2. 환각 누적 (Hallucination Accumulation)

세션이 길어질수록 LLM은 다음과 같이 드리프트한다.

  • 명시된 규칙을 "기억한다"고 주장하지만 실제로는 우회
  • 사용자 의도와 다른 방향으로 작업 확장
  • 존재하지 않는 파일·함수·API 호출
  • 권한 없는 코드 변경

LLM의 "Lost in the Middle" 현상과 결합해 컨텍스트 중반 정보가 소실되면 환각은 기하급수적으로 증가한다.

대응 전략

  • "Hooks > Prompts" 원칙
  • "Single Responsibility per Role" 원칙

3. 토큰 비용 폭발 (Token Cost Explosion)

작은 수정을 위해 큰 파일을 통째로 Read하는 패턴이 만연한다.

  • 20줄 edit을 위해 1,500줄 파일 전체 로드 → ~15,000 토큰 낭비
  • bash `find` / `grep -r` 결과를 컨텍스트에 흡수 → 수천 토큰 낭비
  • "확인용" 반복 Read → 매 턴 수천 토큰 누적

대응 전략

  • Cave-Man Protocol
  • Dictionary.md 좌표 시스템의 BLK 좌표 인덱싱
  • RULE7 PRD Spatial Mapping: EXECUTE 단계에서 좌표 없는 행동 차단

4. 세션 단절 (Session Discontinuity)

새 세션을 열거나 다른 AI(Trae, Gemini)에게 작업을 넘길 때 컨텍스트가 0에서 시작한다. 사용자는 모든 배경을 다시 설명해야 하며, AI마다 동작 규칙이 달라 동일 결과를 보장할 수 없다.

대응 전략

  • 4계층 메모리 아키텍처의 L1·L4 계층
  • External AI 역할분리
  • doc-context 입력 워크플로우

5. 에이전트 디스패치 혼란 (Agent Dispatch Confusion)

어떤 작업에 어떤 에이전트를 호출할지 자체가 복잡한 결정이 된다.

  • 단순 질문에 Opus 에이전트 호출 → 비용 낭비
  • 멀티시스템 작업을 단일 에이전트가 처리 → 품질 저하
  • 검증 단계 누락 → 결함 누적

대응 전략

  • 3Tier 에이전트 스튜디오의 Tier 분리
  • Producer-QualitySentinel-Reporter 게이트 체인
  • 작업 규모별 워크플로: 규모별 라우팅 정의

6. 통찰

이 다섯 문제는 모두 "LLM이 모든 것을 기억하고, 자유롭게 탐색하고, 스스로 판단하면 잘 될 것이다"라는 가정의 실패를 보여준다. 5대 문제의 공통 해법은 AI의 자율성을 제한하고, 상태와 인덱스를 외부에 두는 것이다.
