# L1 방어선 시나리오 X 결정

[[L1_L5_다층방어선_역사]]가 5중에서 1+1+외부로 축소되기 직전, 네 개의 후보 시나리오를 비교해 *시나리오 X*가 채택된 의사결정 과정. 보장률 단계별 측정값과 잔여 탈옥 경로 4종의 분석.

## 보장률 단계별 측정

다층 방어선 운영 직전 수집된 단계별 측정값:

| 활성 계층 | 턴당 추가 토큰 | 보장률 |
|---|---|---|
| L1만 | +20% | 70% |
| L1 + L3 + L5 (표준) | +25% | 92% |
| L1 + L3 + L4 + L5 (강화) | +30% | 97% |
| L1~L5 전체 (full stack) | +32% | **99.2%** |

L1만 단독 운용해도 70%. 텍스트 reminder(L3 + L5)를 더하면 22%p가 추가된다. L4까지 더해 99.2%에 닿지만, *마지막 7%p를 위해 토큰이 비선형*으로 늘어난다 — 한계 효용 체감이 명확하다.

## 후보 시나리오 비교

후속 분석에서 다음 4개 시나리오가 비교됐다:

| 시나리오 | 구성 | 턴당 토큰 | 보장률 |
|---|---|---|---|
| A (현상 유지) | L1~L5 전체 | +827 | 99.2% |
| B (L4만 제거) | L1+L2+L3+L5 | +677 | 98% |
| C (L2+L4 제거) | L1+L3+L5 | +560 | 94% |
| **X (채택)** | **L1+L3, L5 외부 이전** | **+310** | **94%** |

시나리오 C와 X는 보장률이 동일하지만 X는 *매 턴 reminder*를 *cxt 트리거 시점 1회*로 변환해 -250 토큰을 더 회수한다. [[External_AI_역할분리]] 안에서 trae의 context-sharer가 단일 책임을 지므로 책임 분리 원칙 위반도 아니다.

## 시나리오 X의 절충 합리화

```
손실:    보장률 99.2% → 94% (-5.2%p)
획득:    턴당 +827 → +310 (-62%)
정책 파일: 11개 → 5개 (-55%)
```

LLM 본질상 100% 보장은 불가능하므로, *5%p의 보장률을 토큰 62%로 교환*하는 게 합리적 선택이라는 결론. 손실분은 context-sharer 임베드로 부분 보완 ([[L1_L5_다층방어선_역사]] § L5의 외부 이전).

## 잔여 탈옥 경로 4종

99.2%든 94%든 닿지 못하는 잔여 경로:

| 경로 | 설명 | 대처 |
|---|---|---|
| 사용자 명시 우회 | "이번엔 그냥 grep으로 해줘" | 허용 (의도된 우회) |
| 신규 도구 등장 | hook 매처에 없는 새 도구 | hook 패턴 업데이트 |
| Claude Code 자체 버그 | hook 발화 자체 실패 | 버그 리포트 |
| CLAUDE.md 미주입 | SessionStart 실패 시 컨텍스트 결손 | SessionStart 검증 (L3가 차단) |

이 4가지는 [[L1_Hard_Gate_훅체계]]의 hook 만으로 0%로 만들 수 없는 *물리적 한계*다. `violation-count.log` 모니터링으로 패턴이 드러나면 hook 업데이트의 근거가 된다.

## 결정의 정신적 모델

```
1. 결정론적 차단(L1)이 한 일을 텍스트 reminder가 다시 하면 그것은 잉여다.
2. LLM 자가검증 텍스트는 attention drop으로 효과 ~50%에 그친다.
3. 한 번 차단되면 끝인 것에 매 턴 +150 토큰을 쓰는 건 비효율이다.
4. 트리거 시점 1회 reminder(cxt 임베드)가 매 턴 reminder보다 효율적이다.
```

이 네 명제가 *Hook > Prompts*([[LLM_통제_철학_6대원칙]] 원칙 3)의 정량적 재확인이다. 텍스트 reminder의 한계를 토큰 사용량과 함께 본 결정.

## 시나리오 X의 위험과 대응

5%p 보장률 손실의 위험은 다음 모니터링 임계치로 관리한다:

- 위반 7일 누적 > 20회 → L5 복구 검토 (시나리오 C로 회귀)
- 토큰 절감 < 50% → 추가 최적화 검토
- 사용자 개입 +30% 이상 → context-sharer 임베드 강화

세 임계치 중 하나라도 깨지면 결정의 정당성이 재검토되어야 한다. 결정을 영구화하지 않고 *감사 가능한 결정*으로 둔다.

## 시나리오 X의 외부 효과

이 결정이 [[Workflow_Ecosystem_버전계보]] v3.4의 정량 변화 표를 만들었고, [[Workflow_진화_3단계]] 3단계 전환의 절반(축 A)을 차지한다. 본 결정 없이는 3단계의 외부 Wiki 통합 부담(매 턴 +827 토큰 위에 wiki 동기화 작업까지)을 감당할 수 없었다.
