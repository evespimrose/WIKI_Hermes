# 에이전트 자는시간 학습 루프

사람이 자는 동안 에이전트가 *스스로 리서치·학습*하게 만드는 운영 기법. 크론잡으로 정해진 시간에 논문·학술자료·뉴스·레딧·AI 블로그/커뮤니티를 "학습 깊이 1~10" 기준으로 조사·분석·학습 → 일어나면 완료 상태. [[AI_네이티브_멀티에이전트_운영]]의 시간 절약 핵심.

## 메모리 효율과의 결합

방대한 리서치를 *전부 저장하면 메모리가 터지고 에이전트가 멍청해진다*. 그래서 규칙: "분석·학습이 끝나면 옵시디언에 위키화하고 메모리는 항상 최대 효율을 유지하라. 관련 업무 시 키워드로 옵시디언을 검색해 참조하라." 이는 [[4계층_메모리_아키텍처]]의 외부화 + [[컴팩션_생존_전략]]의 메모리 절약과 정확히 같은 원리 — 학습 *결과*만 L5(위키)에 남기고 작업 메모리는 비운다.

## 자가학습 루프로의 확장

이 "수집→정제→위키화→참조" 루프에 *사람의 피드백*을 더하면 자가 개선이 된다. 본 볼트는 이를 헤르메스 *지식 스카우터*로 구체화하려 한다 — 입국심사 판정을 판례로 누적해 스카우팅 방향을 자가 학습(`docs/Try_HermesKnowledgeScout` §판례 단어장). 즉 본 노트의 학습 루프가 [[위키_저장_5대필터]] 게이트와 결합해 *방향성 있는 자율 학습*이 된다.

## 디자인 영역 사례

디자이너 에이전트는 UI 레퍼런스(Dribbble·Awwwards·Mobbin)와 공개 디자인 시스템을 학습하고, '오픈디자인'을 CLI로 연결해 "95% 이상 똑같이 만들 때까지 무한 루프"로 연습한다 — 학습 루프가 *생성 품질 수렴*에도 적용되는 예.
