# AI 네이티브 멀티에이전트 운영

역할별 'AI 직원' 팀을 꾸려 *하나의 회사처럼* 협업시키는 운영법. 발표자는 10명(헤르메스 5 + 클로드 코드 5)을 운영한다. [[하네스_엔지니어링]]의 멀티에이전트 우위를 일상 업무 규모로 구현한 사례.

## 역할 분리 (에이전시 ⨉ 인하우스)

헤르메스 5명 = 외부 에이전시, 클로드 코드 5명 = 인하우스로 분리해 서로 혼선을 막고 집중 환경을 준다. 역할 예: 개발+모더레이터 / 디자인 / 자동화+검증 / 콘텐츠 / 강의보조. 사람은 *기획·감독·디렉션만* 담당 — [[Mastermind_Architecture_Manifesto]]의 인간 Confirm/Deny 원칙과 같다.

## 맥락 공유가 묶는다

다섯 에이전트가 따로 놀지 않는 이유는 *옵시디언 위키로 맥락을 공유*하기 때문이다([[위키_기반_에이전트_맥락공유]]). 도구(Codex·Claude·Gemini)를 교체해도 위키만 있으면 작업을 바로 이어간다 — 매번 업무 설명 불필요. 이것이 "진정한 에이전틱 엔지니어링"이라 불리는 핵심.

## 세션 send — 에이전트 간 대화

특정 에이전트에 "다른 에이전트들에게 이것을 시켜라" 명령하면, 사람 눈에 안 보이게 에이전트끼리 지시·보고하고 최종적으로 그 에이전트가 사람에게 보고한다. [[Agent_Skill_Orchestrator_3요소]]의 에이전트 팀(실험 기능)·[[3Tier_에이전트_스튜디오]]의 위임 구조와 같은 계열. 다만 그 보고가 메인 컨텍스트로 *재주입*될 때의 토큰 비용은 [[서브에이전트_출력압축_원리]]로 통제한다 — 압축하지 않으면 위임이 누적될수록 컨텍스트가 고갈된다.

## 서브에이전트 컨텍스트 격리

여러 에이전트가 따로 노는 *부작용*을 막는 동시에 그 *장점*을 취하는 구현 규약: **태스크당 신규 서브에이전트를 띄워 세션 히스토리 오염을 차단**한다. 한 에이전트가 누적한 잘못된 전제·드리프트가 다음 태스크로 새어 들어가지 않도록, 태스크 경계에서 컨텍스트를 *리셋*하는 것이다 — [[맥락_오염_방지]]를 *세션 히스토리 레벨*에서 구현한 패턴.

비용이 명확하다: 서브에이전트 1개 초기화에 3,000~8,000 토큰이 든다(하네스 의존). 단독·소형 태스크에 매번 새 에이전트를 띄우면 오버킬이다. 따라서 [[작업_규모별_워크플로]]의 "플랜 파일 존재 = 중·대형" 기준을 엄격히 적용한 태스크에만 격리를 권한다 — 품질 보증(오염 차단)의 대가가 초기화 비용을 정당화하는 규모에서만.

## 학습은 별도 투자

헤르메스 설치만으로 실무가 되지 않는다 — 도메인 지식·작업방식을 학습시키는 물리적 시간이 필요하며, 그 시간을 *자는시간 + 크론잡 + 학습 루프*로 절약한다([[에이전트_자는시간_학습루프]]). 도구보다 *적용처와 본인 전문성*이 성패를 가른다.
