# 위키 기반 에이전트 맥락 공유

여러 AI 에이전트가 *하나의 위키를 공유 두뇌로 삼아* 동일 업무 맥락을 나누는 운영 핵심. "좋은 모델을 쓰는 것보다 *모든 에이전트가 동일 맥락을 공유*하게 하는 것이 훨씬 중요하다"가 명제. 이 볼트를 *능동적 신경계*로 진화시키려는 목표의 직접 근거.

## 왜 모델보다 중요한가

개인 업무는 맥락이 날아가면 다시 설명하면 그만이지만, 여러 부서·에이전트가 얽힌 업무에선 치명적이다(밤새 분석시켜 의도와 다른 기획을 내놓는 등). 맥락 공유가 잡혀야 *에이전트가 바뀌거나 세션이 끊겨도* 안정적 결과가 보장된다 — [[카파시_LLM_위키_패턴]]의 RAG vs 위키 차이가 이 보장의 근거.

## 도구 교체를 견디는 자산

클로드 코드·헤르메스·Codex·Gemini 어떤 도구를 쓰든 *위키만 있으면 작업을 이어간다*. 지식이 도구가 아니라 위키에 누적되므로, 도구를 갈아도 자산이 이전된다 — [[4계층_메모리_아키텍처]] L5(영속)·[[External_AI_역할분리]]의 도구 무관 지식 자산화와 동형.

## 이 볼트의 현황과 목표

본 볼트는 이 패턴의 *저장* 측면([[LLM_Wiki_운영체계]])은 완비했으나, *에이전트가 턴/온디맨드로 참조하는 런타임 회로*가 아직 없다([[LLM_Wiki_지식자산_현황]]의 "빠진 마지막 1마일"). 진화 경로는 `docs/Roadmap_NervousSystemEvolution`:
- 프로젝트 세션이 키워드/의도로 행성을 시맨틱 질의 (Track B)
- 헤르메스가 같은 위키를 읽기 공유 + 스카우터로 쓰기

## 오염 위험과 게이트

공유 두뇌는 강력한 만큼 [[맥락_오염_방지]] 위험을 동반한다. 그래서 어떤 에이전트도 위키 직접 쓰기는 [[위키_저장_5대필터]]·입국심사 게이트를 거쳐야 한다 — 공유의 이득과 오염의 위험을 게이트로 가른다.
