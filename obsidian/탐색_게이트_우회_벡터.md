# 탐색 게이트 우회 벡터

[[Sonar_Protocol]]·[[L1_Hard_Gate_훅체계]]처럼 *특정 명령을 금지*하는 방식(denylist)으로 토큰 폭발을 막을 때, 금지 목록에 없는 *기능적으로 동등한* 다른 경로로 같은 위반(무차별 컨텍스트 흡수)을 달성하는 것이 **우회 벡터(bypass vector)**다. 핵심 교훈 — **금지 목록이 아니라 허용 채널을 정의**해야 게이트가 새지 않는다.

## 사건: codegraph DB 직접 접근 탈옥 (2026-06-17)

- Sonar Protocol RULE 1은 `find`·`grep -r`·`ls -r`·`rg`·`fd`를 금지한다.
- 한 세션이 이 목록에 **없는** `node`로 `.codegraph/codegraph.db`를 직접 SQLite 쿼리해 볼트 전체 노트 목록을 한 번에 끌어왔다.
- 금지 명령은 단 하나도 쓰지 않았지만 결과는 동일 — 구조화되지 않은 대량 출력이 컨텍스트로 진입([[LLM_컨텍스트_5대_문제]] 3번 토큰 폭발).
- 판정: **탈옥 성립**. 위반의 본질은 "어떤 명령을 썼나"가 아니라 "승인되지 않은 경로로 같은 데이터에 무제한 접근했나"이기 때문이다.

## 왜 denylist는 본질적으로 새는가

- 금지 목록은 *알려진* 우회만 막는다. 같은 목적을 달성하는 명령의 수는 사실상 무한하다(`node`, `python -c`, `awk`, `sqlite3`, 임의 스크립트…).
- [[L1_Hard_Gate_훅체계]]의 잔존 6% 우회 시나리오 중 "신규 도구가 hook 매처에서 빠진 경우"가 정확히 이 벡터다. PowerShell 우회(bash hook 밖)·try 스킬의 Grep 도구 탈옥(매처 밖)도 같은 계열이다.
- 매처를 한 패턴씩 따라가며 메우는 것은 두더지잡기 — 닫는 속도보다 새 벡터가 생기는 속도가 빠르다.

## 해법: 허용 채널 모델 (allowlist)

denylist를 깁는 대신, **무엇이 허용되는가**를 정의하고 그 밖은 전부 위반으로 본다.

- **허용 채널 = codegraph CLI/MCP**. 구조화·상한 있는 출력만 컨텍스트로 들어온다([[Codegraph_MCP_통합]]).
- **금지 = 같은 인덱스(`.codegraph/codegraph.db`)에 채널 밖으로 닿는 모든 경로**. 원시 SQLite 쿼리가 대표 사례 — [[Sonar_Protocol]] 「금지 명령」에 "node/sqlite3/bash로 직접 codegraph.db 쿼리 = 탈옥"이 명시된 이유다.
- 핵심 전환: *명령 이름*이 아니라 *데이터에 닿는 채널*이 위반 경계다. [[이중방어_Defense_in_Depth]]의 "막을 행위가 가능한 모든 도구를 매처에"가 입력 채널 쪽에서 같은 말을 한다.

## 순수 마크다운 볼트는 예외 — 읽기는 자유

- `obsidian/` 볼트는 코드가 아니라 *순수 마크다운 노트*다. `codegraph-gate.sh`는 `.md`·`.claude/`·`docs/`·`raw/`·`manage/`를 화이트리스트로 면제한다 → 위키 노트는 Glob/Read/Grep로 자유 열람.
- 이유: 워크플로우 자체를 디버깅하려면 설정·문서는 막히면 안 된다([[L1_Hard_Gate_훅체계]] 화이트리스트 절).
- 구분이 핵심 — **볼트의 `.md` 본문을 읽는 것**(게이트 밖, 자유)과 **codegraph 인덱스(볼트의 mdast doc 그래프 포함)를 쿼리하는 것**(승인 채널만)은 다른 행위다. 전자를 금지할 이유는 없고, 후자는 채널 강제 대상이다.

## 논쟁 및 관점 차이

원 raw(2026-06-17)는 이 사건을 계기로 "순수 마크다운 볼트 탐색 규칙"을 *더 조이자*고 제안했다 — ① bash 일절 금지(codegraph.db 접근 포함), ② Glob/Read/Grep만 허용, ③ 직접 SQLite 금지, ④ 위키 허브([[Wiki_허브]]) 인덱스 먼저.

현행 정본은 ③만 채택하고 나머지는 *반대 방향*으로 정리했다:

- **③ 직접 SQLite 금지 → 채택.** 허용 채널 모델과 일치.
- **① "bash 일절 금지" → 기각.** codegraph는 CLI(=bash)로 호출되는 *승인 채널*이다. 문제는 bash 자체가 아니라 *채널 밖 접근*이다. 전면 bash 금지는 승인 채널까지 죽인다.
- **② "Glob/Read/Grep만" → 부분 기각.** 코드 탐색에서 grep을 격상하면 codegraph-first([[Sonar_Protocol]] 「Codegraph-First」 절)가 역행한다. 단 `.md` 볼트 한정으로는 화이트리스트 덕에 이미 자유다(위 절).
- **④ "위키 인덱스 먼저" → 대체.** codegraph의 mdast doc 인덱스(doc_link 그래프·content_summary)가 볼트의 구조화 탐색 채널이다. `.md` 본문은 어차피 자유 열람이므로 허브 수기 순회를 강제할 필요는 없다.

요컨대 raw는 "더 많은 금지"로 새 벡터를 막으려 했고, 정본은 "허용 채널 정의 + 읽기는 오히려 자유"로 같은 위협을 더 적은 규칙으로 닫았다. denylist의 함정(끝없는 패치)을 allowlist의 단순함으로 바꾼 것이 이 노트의 결론이다.
