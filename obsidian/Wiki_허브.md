# Wiki 허브

LLM Wiki vault 자체의 운영 정책 노트들을 묶는 행성. 다른 두 행성과 달리 *vault 자체에 대한 메타 정책* 영역이다. 상위 항성 [[LLM_워크플로우_생태계]]에서 진입한다. 인접 행성은 [[Token_허브]]·[[RX_1_허브]].

## 행성 정의

[[Token_허브]]가 *LLM 통제 도구*, [[RX_1_허브]]가 *프로젝트 인스턴스*라면, 본 행성은 *그 모든 지식이 어디에 어떻게 저장·동기화되는가*를 다룬다. 작은 별자리지만 다른 두 행성의 *물리적 받침대* 역할.

## 위성 카탈로그

### 운영 체계

- [[LLM_Wiki_운영체계]] — vault 구조, 3단계 파이프라인 (move-to-raw → compile-wiki → archive), 정책 원칙

### 동기화 인프라

- [[Git_기반_다중PC_동기화]] — git 저장소 구조, .gitignore 정책, 다중 PC 워크플로우

### 자산 현황

- [[LLM_Wiki_지식자산_현황]] — 노트 맵핑, 진화 방향 (Self-Healing, Mastermind, Multi-Vault)

### 카파시 위키 패턴 (벤치마크·자기서술)

- [[카파시_LLM_위키_패턴]] — 본 볼트가 구현하는 메타 패턴 (RAG vs 위키)
- [[Raw_Wiki_Schema_3레이어]] — Raw 불변 / Wiki 가공 / Schema 규칙
- [[맥락_오염_방지]] — Context Pollution, 오염 전파 차단
- [[위키_저장_5대필터]] — 위키 승격 게이트 (입국심사 이론 토대)
- [[위키_기반_에이전트_맥락공유]] — 위키를 다중 에이전트 공유 두뇌로

## 별자리의 성장

처음 위성 3개로 출발했으나, 카파시 위키 패턴([4] 유입)을 편입하며 8개로 늘었다. 본 행성은 이제 *vault 운영 정책*을 넘어 *위키라는 것 자체의 이론*까지 담는다 — 볼트가 자기 자신을 서술하는 메타 영역. 신경계 진화(`docs/Roadmap_NervousSystemEvolution`)의 Schema 배선이 더해지면 계속 성장한다.
