# 압축 효과 측정 정직성

압축 % 주장의 정직성은 **"무엇 대비 측정인가"** 한 문장으로 결정된다. caveman의 eval은 3-arm이다 — `__baseline__`(무프롬프트) / `__terse__`("Answer concisely.") / `<skill>`. **정직한 델타 = skill vs terse**다. 장황 baseline 대비로 재면 스킬을 *generic 간결*과 혼동해 숫자가 부푼다(레포 자인: earlier version이 정확히 그래서 inflated였다).

## 함의 — 절감의 대부분은 스킬 고유가 아니다

어떤 압축 스킬이든 "장황 대비 −65%"의 상당부는 "간결히"라는 generic 지시가 만든다. 스킬의 *고유 기여*는 terse 대조군 대비 마진뿐이다. 따라서 압축 도구를 평가할 땐 항상 terse arm을 끼워야 한다 — 안 그러면 도구 효과와 "그냥 짧게 답해"를 분리할 수 없다.

## 측정 함정 3종

1. **근사 토크나이저** — caveman은 `tiktoken o200k_base`(OpenAI BPE)로 Claude 토큰을 *근사*한다. arm 간 비율만 유의하고 절대수는 부정확. 다른 모델의 토크나이저로 잰 %는 자기 모델 %가 아니다.
2. **n=1·기본 temp·non-powered** — 프롬프트당 단일 표본. 변동이 커(caveman 범위 22–87%) 평균 한 숫자로 단정하면 안 된다.
3. **fidelity 미측정** — % 절감은 정보 *보존*을 보장하지 않는다. caveman eval 자인: `k`라고만 답하면 −99%로 '승리'한다. 절감률이 정확도를 *못* 본다.

## 볼트에서의 위치

본 볼트도 [[Sonar_Protocol]] −62%, [[출력_표현_압축]] 순 −30%/턴 같은 % 주장을 다수 보유한다. 외부 인용·자기 보고 시 **대조군(무엇 대비)·표본수·토크나이저**를 명시해야 정직하다 — [[Token_허브]] 토큰 통제 카탈로그의 측정 규약. 절감률의 *정확도 차원*은 [[압축도구_표방실제_검증]]이 짝으로 다루고, "어디까지 줄여도 안전한가"의 상한은 [[압축_안전경계]]가 맡는다.
