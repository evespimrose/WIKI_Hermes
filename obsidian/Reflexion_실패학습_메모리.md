# Reflexion 실패학습 메모리

과거의 *실패*를 구조화해 누적하고, 다음 세션이 그것을 즉시 회피하게 만드는 학습 루프. [[4계층_메모리_아키텍처]] L3(장기기억)의 하위 패턴으로, 기존 `memory:save/recall`이 *지시·사실·결정 내러티브*를 운반한다면 이쪽은 *실패→예방규칙* 변환을 전담한다. 볼트가 가진 적 없는 메커니즘이다.

## 변환 루프

```
mistake → evidence → rule → fix → tests → status:adopted
```

한 번의 실패를 6단계로 박제한다: 무엇을 틀렸나(mistake) → 근거(evidence) → 일반화한 예방규칙(rule) → 적용한 수정(fix) → 재발 검증(tests) → 채택 상태(status). JSONL 한 줄 = 한 판례. 다음 세션은 작업 착수 전 이 규칙들을 조회해 *같은 함정을 두 번 밟지 않는다*.

## 왜 L3의 별도 슬롯인가

[[4계층_메모리_아키텍처]]의 L3는 "무엇을 왜 결정했나"의 *의미 메모리*다. 실패 규칙은 그 안의 특수 슬롯 — "무엇을 하지 *말아야* 하나"의 음(陰)의 기억이다. [[위키_저장_5대필터]]의 4번(실패 회피성)·[[Graveyard_안티패턴_원칙]]이 "드문 고신호인 실패를 남긴다"고 *정책*으로 말한 것을, Reflexion은 *세션 간 자동 루프*로 실행한다.

## 인접 학습 루프와의 구분

[[에이전트_자는시간_학습루프]]가 *리서치→정제→위키화*의 **수집형** 학습이라면, Reflexion은 *실패→규칙*의 **교정형** 학습이다. 둘은 같은 "외재화 후 다음 세션이 참조" 골격([[컴팩션_생존_전략]])을 공유하되 입력이 다르다 — 한쪽은 새 지식, 한쪽은 자기 오류.

## 채용 시 경계

- 런타임 비용 0 — 개념·스키마 추출이며, 실제 JSONL 검색은 <1MB에서 무시 가능.
- 유일한 리스크: evidence에 비밀정보(API키·비밀번호)가 섞이면 수기 레닥션이 필요하다. 원천도 동일 경고를 명시한다.
