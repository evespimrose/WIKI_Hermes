# 결정론 검증 게이트

비결정적 LLM 연산의 산출물을 **결정론적 구조 불변식**으로 감싸 검증한다. 패턴: `LLM이 변환(비결정) → 코드가 구조 불변식 대조(결정론) → 실패 시 cherry-pick 재시도 ≤N → 그래도 실패면 원본 롤백`. LLM을 신뢰하되 *검증은 코드가* 한다 — [[LLM_통제_철학_6대원칙]]의 "Hooks > Prompts"를 파일·산출물 레벨로 구현한 사례.

## 구조 불변식의 예 (caveman-compress validate.py)

LLM이 마크다운을 압축한 뒤, 코드가 다음을 *정확 대조*한다:
- 제목 수 일치 / 코드블록 바이트 정확 일치 / URL 집합 동일 / inline code 보존 / bullet 수 ±15% / 경로 보존.
- 위반 시 누락 항목만 짚어 재압축이 아닌 **타깃 수정**을 요청(≤2회), 끝내 실패하면 원본 복원·백업 삭제.

즉 "LLM이 약속을 지켰는지"를 프롬프트 신뢰가 아니라 *기계 검사*로 판정한다.

## 또 다른 사례 — Headroom 압축 불변식

같은 패턴(비결정 LLM 변환을 결정론 불변식으로 감싸기)이 입력 *내용* 압축에서도 나타난다. Headroom의 압축 파이프라인은 다음과 같은 불변식 세트를 강제한다:

- **byte-equal**: 미변경 영역은 SHA-256이 정확히 일치해야 한다.
- **append-only**: live zone에만 추가하고, 이미 굳은 영역은 수정하지 않는다.
- **hot-zone freeze**: system·tools·과거 턴·reasoning 영역은 불변으로 동결한다(프리픽스 안정화 — [[LLM_통제_철학_6대원칙]] §1 KV 캐시와 연동).
- **결정성**: 같은 입력 → 같은 출력. tool 정의는 정규화하되 무압축으로 보존.

검증 방식이 핵심이다 — *parity harness*로 Rust/Python 두 구현의 바이트 패리티를 회귀 테스트한다. 즉 "결정론은 선언이 아니라 *테스트로 강제*한다." caveman의 validate.py가 *산출물 1건*을 사후 대조한다면, 이 불변식 세트는 *파이프라인 전체*를 사전·회귀로 묶는 안전 계약(contract) 템플릿이다. LLM이 개입하는 모든 변환 파이프라인의 설계 참조가 된다.

## 핵심 한계 — 구조 ≠ 의미

이 게이트는 **구조** 불변식만 검사한다. 산문의 *의미*는 LLM이 rewrite하므로, 의미 드리프트가 구조 검증을 통과할 수 있다("구조 100%"이지 "의미 100%"가 아니다). 따라서 헌법급 문서(CLAUDE.md·schema)는 LLM lossy 압축 대상에서 제외하고, 결정론(수기·규칙 기반) 압축만 허용해야 한다.

## 볼트에서의 위치

[[Producer_QualitySentinel_Reporter_게이트]]가 에이전트 *체인*을 호출 그래프로 강제하듯, 본 게이트는 단일 *산출물*을 불변식으로 강제한다 — 같은 철학의 다른 입도. [[L1_Hard_Gate_훅체계]]·[[이중방어_Defense_in_Depth]]의 "통과 못 하면 진행 불가" 구조를 LLM 변환 파이프라인에 적용한 것. [[위키_저장_5대필터]] 승격 게이트도 이 패턴으로 자동화 가능(단 구조≠의미 한계 명기). 같은 "코드가 한다" 철학의 입력측 형제 — 코드가 *검증*하는 본 게이트와 달리 코드가 *압축*하는 [[MCP_도구카탈로그_결정론압축]].
