# LLM Wiki 운영 체계

중앙 지식 저장소의 입출력 파이프라인, Vault 구조, 핵심 정책 원칙을 정리한 운영 체계.

## Vault 구조: 3개 폴더의 역할 분담

```
D:\Fork\WIKI/
├── raw/        ← 입수 단계: 텍스트성 자료 임시 수용
├── obsidian/   ← 컴파일 단계: 원자적 위키 노트 (최종 저장)
└── archive/    ← 보관 단계: 처리 완료 원본 (감사용)
```

| 폴더 | 특성 | 책임 |
|---|---|---|
| `raw/` | 쓰기 가능 | 모든 외부 자료가 먼저 이곳에 떨어짐 (move-to-raw 스킬) |
| `obsidian/` | 생성 전용 | 강하게 상호 연결된 지식 그래프 (compile-wiki 스킬) |
| `archive/YYYY-MM/` | 읽기 전용 | 월별 보관. 재처리나 감사 검증용 |

## 3단계 파이프라인

### Phase 1: 입수 (move-to-raw)

PC 임의 위치의 텍스트 자료를 `raw/` 폴더로 모으는 단계.

**역할**: [[move-to-raw]] 스킬이 자동 수행
- 파일 경로 확인 및 텍스트 필터링 (바이너리 거부)
- 이름 충돌 해결 (출처 디렉토리 접미사, 타임스탬프)
- 기본값: **복사** (원본 손상 0)
- 예외: 사용자가 "옮겨줘" 표현 시만 **이동**

**결과**: raw/ 폴더에 준비된 자료 목록

### Phase 2: 컴파일 (compile-wiki)

raw 자료를 원자적 노트로 분해하고 obsidian/에 구축하는 단계.

**역할**: [[compile-wiki]] 스킬이 자동 수행

**2-1. 원자성 (Atomic Notes)**
- 한 노트 = 한 개념만
- 명사형 파일명 (예: `Dictionary_md_좌표_시스템.md`)
- 공백은 언더바로

**2-2. 표준 구조**
```yaml
---
tags: [llm-wiki, 분류태그1, 분류태그2]
date_created: 2026-05-26
last_modified: 2026-05-26
---
```

**2-3. 양방향 링크**
- 본문에서 기존 노트명 발견 → `[[노트명]]`으로 링크
- 역으로 기존 노트도 수정해 새 노트 가리키기
- 고립 노트 금지: 모든 노트는 최소 1개 다른 노트와 연결

**2-4. 추상화**
- 원본 토큰의 30~50% 수준으로 요약
- 원본 복사-붙여넣기 금지
- 출처는 `## 출처` 섹션에 명시

**결과**: obsidian/ 폴더의 새로운 노트들

### Phase 3: 보관 (archive)

컴파일 완료된 원본을 월별로 저장하는 단계.

**이동 조건**:
- 해당 raw 파일에서 추출된 모든 노트가 성공적으로 생성됨
- 사용자가 결과를 검수함

**디렉토리 구조**:
```
raw/foo.md           → archive/2026-05/foo.md
raw/unity/bar.md     → archive/2026-05/unity/bar.md
```

**보존 원칙**:
- archive의 파일은 절대 수정 금지
- 원본 폴더 계층 유지
- 동명 파일 충돌 시 `_2`, `_3` 접미사 추가

**결과**: 감사 추적이 가능한 원본 보관소

## 핵심 정책 4가지

### 1. 신뢰성 (Reliability)

**원본 보존**: archive로 이동된 자료는 절대 변경 금지. 재처리나 결과 검증 시 필요.

**동기화 보증**: [[Git_기반_다중PC_동기화]]를 통해 obsidian/ 최신 상태가 모든 PC에 반영.

**중복 방지**: move-to-raw의 이름 충돌 해결 메커니즘으로 같은 자료 중복 입수 차단.

### 2. 가용성 (Accessibility)

**단일 진입점**: 모든 자료는 raw/ 한 곳으로 수렴. 
- PC 어디서든 자료 추가 가능
- move-to-raw 스킬이 경로 자동 관리

**마찰 최소화**: vault 경로 일회 설정 후 반복 질문 없음.

### 3. 확장성 (Scalability)

**원자 설계**: 각 노트의 독립성 + 링크 연결로 대량 데이터도 관리 가능.

**프로젝트 인스턴스**: [[RX_1_manage_DB_스키마]]처럼 특정 프로젝트 특화 노트도 일반 그래프에 자연스럽게 통합.

**레이어 계층화**: obsidian/ 내 폴더 구조 + 월별 archive 관리로 확장성 확보.

### 4. 검사 추적성 (Auditability)

**출처 명시**: 모든 노트의 `## 출처` 섹션에 유입 원본 기록.

**archive 보존**: 원본 파일을 월별로 보관해 생성 이력 추적 가능.

**git 기록**: obsidian/ 변경 사항이 git 커밋으로 남아 언제 어떤 노트가 생성/수정되었는지 추적.

## 운영 원칙의 철학적 근거

이 3단계 파이프라인은 [[LLM_통제_철학_6대원칙]]의 다음 원칙들을 실행한다:

- **원칙 1** (토큰은 1급): dictionary 압축 + BLK 인덱싱으로 토큰 절감
- **원칙 2** (상태 외부화): 지식 상태를 obsidian/로 완전 외부화
- **원칙 3** (훅 > 프롬프트): 스킬 자동화로 결정론적 파이프라인 강제
- **원칙 5** (단일 책임): move-to-raw, compile-wiki가 각각 하나의 책임만 수행
