# 4계층 메모리 아키텍처

[[LLM_통제_철학_6대원칙]]의 "Externalize State" 원칙을 구체화한 계층형 메모리 시스템. 각 계층은 서로 다른 시간 척도와 갱신 주기를 가지며, 어떤 계층을 단순화하면 다른 계층의 부담이 늘어난다. 4계층 분리가 *각 계층의 단순성을 보장*한다.

## L1 — 세션 컨텍스트 (휘발성)

| 속성 | 값 |
|---|---|
| 위치 | `CLAUDE.md` + `session-start.sh` 훅 |
| 갱신 주기 | 매 세션 |
| 보존 메커니즘 | 훅이 매 세션 자동 재주입 |
| 풀어주는 문제 | 매 세션 깨끗한 상태로 시작 |

매 세션이 0에서 시작하는 [[LLM_컨텍스트_5대_문제]]의 4번(세션 단절) 일부를 직접 해결한다.

## L2 — 모드 상태 (영속, 파일 기반)

| 속성 | 값 |
|---|---|
| 위치 | `.claude/memory-bank/.riper-state` |
| 갱신 주기 | 모드 전환마다 |
| 보존 메커니즘 | 디스크 파일 |
| 풀어주는 문제 | 컴팩션 시 [[RIPER_5단계_상태머신]] 모드를 잃는 문제 |

RIPER의 상태머신은 LLM 컨텍스트 안에 있으면 컴팩션 절벽에서 즉시 사라진다. 파일에 박제해 컴팩션과 독립적으로 유지한다.

## L3 — 장기 기억 (압축 전후 보존)

| 속성 | 값 |
|---|---|
| 위치 | `/memory:save` · `/memory:recall` (claude-mem 플러그인) |
| 갱신 주기 | 컴팩션 전 수동 |
| 보존 메커니즘 | 브랜치별 메모리 뱅크 |
| 풀어주는 문제 | 깊은 맥락을 다음 세션에 전달 |

의미 메모리(semantic memory)로서, 무엇을 어떤 이유로 결정했는지의 *내러티브*를 다음 세션에 운반한다. [[컴팩션_생존_전략]]과 짝을 이룬다.

## L4 — 구조 인덱스 (프로젝트 영속)

| 속성 | 값 |
|---|---|
| 위치 | `manage/dictionary.md` (BLK 코드 매핑) |
| 갱신 주기 | 파일 추가·이동·삭제 시 |
| 보존 메커니즘 | git 추적 파일 |
| 풀어주는 문제 | "그 파일이 어디 있더라"의 [[LLM_컨텍스트_5대_문제]] 3번(토큰 폭발) |

L4의 구체 구조는 [[Dictionary_md_좌표_시스템]]에서 §0~§7 별로 다룬다. L4는 [[BLK_좌표_시스템]]을 통해 [[Sonar_Protocol]]을 가능하게 한다.

## 계층 간 시간 척도

```
L1 (세션) →→ L2 (모드 전환) →→ L3 (컴팩션) →→ L4 (파일 변경)
   짧음                                              김
```

각 계층은 *자기 시간 척도의 변화만* 책임진다. L1이 L4의 일을 떠맡으면 매 세션 거대한 인덱스를 재주입하게 되어 토큰이 폭발한다. L4가 L1의 일을 떠맡으면 매번 깃 커밋이 필요해진다.
